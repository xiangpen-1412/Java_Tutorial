## 1. Core Concept: What is Random in Java?

In Java, most "random" numbers are actually:

> **Pseudo-Random Numbers (PRNG)**

This means:

- numbers are generated by a deterministic algorithm
    
- given the same seed → same sequence
    
- not truly random
    

Core model:

```text
next = f(previous)
```

---

## 2. `Math.random()`

### 2.1 Definition

```java
Math.random()
```

Returns:

```text
[0.0, 1.0)
```

### 2.2 Usage

```java
double r = Math.random();
```

### 2.3 Generate Range

```java
int x = (int)(Math.random() * 10);      // 0~9
int x = (int)(Math.random() * 10) + 1;  // 1~10
```

```java
int min = 5, max = 15;
int x = (int)(Math.random() * (max - min + 1)) + min;
```

### 2.4 Notes

- internally uses a shared `Random`
    
- no seed control
    
- not suitable for production logic
    

---

## 3. `java.util.Random`

## 3.1 Basic Usage

```java
Random r = new Random();
```

---

## 3.2 Common Methods

```java
r.nextInt();        // full int range
r.nextInt(10);      // 0~9
r.nextDouble();     // [0,1)
r.nextBoolean();
r.nextLong();
```

---

## 3.3 Range Formula (IMPORTANT)

```java
int x = r.nextInt(max - min + 1) + min;
```

---

## 3.4 Seed Mechanism

```java
Random r = new Random(123);
```

Same seed → same sequence

```java
Random r1 = new Random(42);
Random r2 = new Random(42);

System.out.println(r1.nextInt());
System.out.println(r2.nextInt());
```

---

## 3.5 Internal Principle

Random uses:

> Linear Congruential Generator (LCG)

```text
next = (a * current + c) % m
```

Characteristics:

- deterministic
    
- predictable
    
- not secure
    

---

## 4. `ThreadLocalRandom`

## 4.1 Motivation

`Random` has contention in multi-threading.

---

## 4.2 Usage

```java
ThreadLocalRandom.current().nextInt(10);
```

Range:

```java
ThreadLocalRandom.current().nextInt(5, 11); // 5~10
```

---

## 4.3 Characteristics

- per-thread instance
    
- no locking
    
- high performance
    

---

## 5. `SecureRandom`

## 5.1 Overview

```java
SecureRandom sr = new SecureRandom();
```

---

## 5.2 Usage

```java
int x = sr.nextInt();
byte[] bytes = new byte[16];
sr.nextBytes(bytes);
```

---

## 5.3 Characteristics

- cryptographically secure
    
- unpredictable
    
- slower
    

---

## 6. Comparison

|API|Thread Safe|Performance|Random Quality|Use Case|
|---|---|---|---|---|
|Math.random|Yes|Medium|Medium|demo|
|Random|No|Medium|Medium|general|
|ThreadLocalRandom|Yes|High|Medium|concurrent|
|SecureRandom|Yes|Low|High|security|

---

## 7. Common Pitfalls

### 7.1 Off-by-One

```java
r.nextInt(10); // 0~9
```

---

### 7.2 Wrong Formula

```java
(min) + r.nextInt(max - min + 1)
```

---

### 7.3 Recreating Random

```java
for (...) {
    Random r = new Random();
}
```

→ causes repeated values

---

### 7.4 Security Misuse

Do NOT use `Random` for:

- passwords
    
- tokens
    

Use `SecureRandom`

---

## 8. Example

```java
import java.util.*;
import java.util.concurrent.ThreadLocalRandom;
import java.security.SecureRandom;

public class RandomDemo {

    public static void main(String[] args) {

        System.out.println(Math.random());

        Random r = new Random();
        System.out.println(r.nextInt(10));

        int t = ThreadLocalRandom.current().nextInt(1, 7);
        System.out.println(t);

        SecureRandom sr = new SecureRandom();
        System.out.println(sr.nextInt());
    }
}
```

---

## 9. Interview Key Points

- Random is pseudo-random
    
- based on seed + deterministic algorithm
    
- same seed → same sequence
    
- not suitable for security
    
- ThreadLocalRandom solves concurrency issue
    
- SecureRandom for security-critical scenarios
    

---

## 10. Summary

- Random = deterministic pseudo-random
    
- choose API based on scenario
    
- understand range formula and seed
    
- avoid common pitfalls
