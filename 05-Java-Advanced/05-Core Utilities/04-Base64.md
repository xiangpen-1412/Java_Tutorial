# Base64

## 1. What is Base64?

Base64 is an encoding method that converts binary data into readable text.

In Java, binary data is usually stored as:

```java
byte[]
```

But many places are easier or safer to handle as text, for example:

- URL
- JSON
- HTTP request body
- email content
- database string field

So Base64 is often used like this:

```text
binary data / byte[]
        ↓
Base64 encoding
        ↓
String
```

Example:

```java
byte[] bytes = new byte[32];
secureRandom.nextBytes(bytes);

String token = Base64.getEncoder().encodeToString(bytes);
```

The result is a string that represents the original binary data.

---

## 2. Why is it called Base64?

Base64 uses 64 characters to represent data:

```text
A-Z
a-z
0-9
+
/
```

That is:

```text
26 + 26 + 10 + 2 = 64
```

So it is called Base64.

---

## 3. Base64 is not encryption

This is very important.

Base64 is only encoding, not encryption.

That means:

```text
Base64 can be decoded back easily.
```

Example:

```java
String encoded = Base64.getEncoder().encodeToString("hello".getBytes());
byte[] decoded = Base64.getDecoder().decode(encoded);
String original = new String(decoded);
```

So never treat Base64 as a security mechanism.

Wrong understanding:

```text
Base64 = secure encryption
```

Correct understanding:

```text
Base64 = binary-to-text conversion
```

---

## 4. Common Base64 encoders in Java

Java provides Base64 in:

```java
java.util.Base64
```

There are three common encoders.

---

## 4.1 Standard Base64 encoder

```java
String result = Base64.getEncoder().encodeToString(bytes);
```

This may generate characters like:

```text
+
/
=
```

Example result:

```text
abc+/def==
```

This is fine for normal text storage, but not always ideal for URLs.

---

## 4.2 URL-safe Base64 encoder

```java
String result = Base64.getUrlEncoder().encodeToString(bytes);
```

URL-safe Base64 replaces special characters:

```text
+  ->  -
/  ->  _
```

So the result is safer to put inside a URL.

This is commonly used for:

- password reset tokens
- email verification tokens
- JWT
- temporary access links

Example:

```java
byte[] bytes = new byte[32];
secureRandom.nextBytes(bytes);

String resetToken = Base64.getUrlEncoder()
        .withoutPadding()
        .encodeToString(bytes);
```

---

## 4.3 MIME Base64 encoder

```java
String result = Base64.getMimeEncoder().encodeToString(bytes);
```

MIME Base64 is mainly used in email-related scenarios.

It may insert line breaks automatically because old email systems had line-length limits.

It is commonly used for:

- email attachments
- encoded email body parts
- older MIME-based message formats

---

## 5. What is padding?

Base64 output sometimes ends with:

```text
=
==
```

This is called padding.

Base64 processes data in groups. If the data length does not perfectly fit the group size, Base64 uses `=` to fill the missing part.

Example:

```text
abc==
```

For tokens, padding is usually unnecessary.

So we often write:

```java
String token = Base64.getUrlEncoder()
        .withoutPadding()
        .encodeToString(bytes);
```

This removes the ending `=` characters.

---

## 6. encodeToString()

This method converts a `byte[]` into a Base64 `String`.

```java
String encoded = Base64.getEncoder().encodeToString(bytes);
```

Input:

```java
byte[]
```

Output:

```java
String
```

This is the most common method because tokens, JSON fields, URLs, and HTTP values usually need strings.

---

## 7. encode()

`encode()` also performs Base64 encoding, but it returns a `byte[]` instead of a `String`.

```java
byte[] encodedBytes = Base64.getEncoder().encode(bytes);
```

Input:

```java
byte[]
```

Output:

```java
byte[]
```

This is less common in normal business code.

---

## 8. Decoding

Encoding can be reversed.

Standard decoding:

```java
byte[] decoded = Base64.getDecoder().decode(encodedString);
```

URL-safe decoding:

```java
byte[] decoded = Base64.getUrlDecoder().decode(encodedString);
```

MIME decoding:

```java
byte[] decoded = Base64.getMimeDecoder().decode(encodedString);
```

You should use the matching decoder for the encoder you used.

---

## 9. Common use cases

### Password reset token

```java
byte[] bytes = new byte[32];
secureRandom.nextBytes(bytes);

String resetToken = Base64.getUrlEncoder()
        .withoutPadding()
        .encodeToString(bytes);
```

This creates a random URL-safe token.

---

### JWT

JWT uses Base64Url encoding.

A JWT looks like this:

```text
header.payload.signature
```

The `header` and `payload` parts are Base64Url encoded.

---

### Image as string

```java
byte[] imageBytes = Files.readAllBytes(Path.of("avatar.png"));
String base64Image = Base64.getEncoder().encodeToString(imageBytes);
```

Sometimes the result is used like this:

```text
data:image/png;base64,...
```

---

### Basic Auth

HTTP Basic Auth encodes this format:

```text
username:password
```

using Base64.

But again, Base64 is not encryption.

---

## 10. Summary

Base64 is used to convert binary data into text.

You should remember these three versions:

```java
Base64.getEncoder()       // normal Base64
Base64.getUrlEncoder()    // URL-safe Base64
Base64.getMimeEncoder()   // email/MIME Base64
```

For password reset tokens, the common choice is:

```java
Base64.getUrlEncoder()
        .withoutPadding()
        .encodeToString(bytes);
```

Because the final string is:

- random
- URL-safe
- clean
- suitable for links
