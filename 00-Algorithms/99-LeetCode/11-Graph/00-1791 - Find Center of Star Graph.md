# LeetCode 1791：Find Center of Star Graph

> 难度：Easy  
> 主题：无向图、边列表、度、星型图

## 1. 题目在问什么？

题目给出一张无向星型图的所有边。星型图有一个中心节点，其他每个节点都只和中心节点相连。

```text
    2
    |
3 — 1 — 4
```

边列表可能是：

```text
edges = [[1,2], [3,1], [1,4]]
```

要求返回中心节点 `1`。

题目已经保证输入一定是合法星型图，因此不需要判断图是不是星型图，也不需要遍历整张图验证所有节点。

## 2. 输入容易误读在哪里？

`edges` 是边列表，不是邻接矩阵，也不是二维网格。

```text
edges[i] = [u, v]
```

表示无向边 `u — v`。因为是无向图，`[u, v]` 不表示谁指向谁。

## 3. 核心观察

中心节点会出现在每一条边中。只看前两条边，它们唯一共同的端点就一定是中心：

```text
[1, 2]
[3, 1]
共同端点：1
```

因此问题不需要建邻接表，也不需要 DFS / BFS。

## 4. 两种思路的关系

### 利用星型图保证

比较前两条边的端点，找共同值即可，时间和额外空间都是 `O(1)`。

### 统计每个节点的度

遍历所有边，统计每个节点出现次数。中心节点的度是 `n - 1`，其他节点的度都是 `1`。

这种方法更通用，但没有充分利用题目保证，工作量是 `O(E)`。

## 5. 这题背后能学到什么？

```text
先读清输入结构：int[][] 不一定是网格，也可能是边列表。
先使用题目保证：合法星型图让“前两条边的公共端点”成为充分信息。
不要一看到 Graph 就建图和遍历；有时边本身已经包含答案。
度表示一个节点连接了多少条边，中心节点的度最大。
```

## 6. 一句话总结

> 星型图所有边都经过中心，所以前两条边唯一相同的端点就是答案。
