# LeetCode 1971：Find if Path Exists in Graph

> 难度：Easy  
> 主题：无向图、连通性（connectivity）、连通分量（connected component）

## 1. 题目在问什么？

给定：

- `n` 个节点，编号是 `0` 到 `n - 1`；
- `edges`，每个 `edges[i] = [u, v]` 表示节点 `u` 与 `v` 有一条**双向边**；
- 起点 `source` 和终点 `destination`。

要判断：能否从 `source` 沿着边走到 `destination`。

```text
n = 5
edges = [[0,1], [1,2], [3,4]]

0 —— 1 —— 2      3 —— 4
```

上图有两个连通分量：

```text
{0, 1, 2}
{3, 4}
```

两个节点在同一个连通分量中，就一定存在路径；不在同一个连通分量中，就不存在路径。

## 2. 输入 `edges` 是什么结构？

`edges` 的类型是 `int[][]`，但它不是二维地图，而是**边列表（edge list）**。

```java
int[][] edges = {{0, 1}, {1, 2}, {3, 4}};
```

每个长度为 2 的数组代表一条边：

```text
[0, 1]：0 与 1 相连
[1, 2]：1 与 2 相连
[3, 4]：3 与 4 相连
```

## 3. 两种正确思路

### 思路 A：邻接表 + DFS / BFS

把边列表转换成邻接表：

```text
0: [1]
1: [0, 2]
2: [1]
3: [4]
4: [3]
```

再从 `source` 开始遍历，若访问到 `destination`，说明两点连通。

适合：需要从某个节点出发，逐步查看它的邻居；也适合网格题、路径搜索题。

### 思路 B：并查集（Union-Find）——不建邻接表

逐条读取原始 `edges`：

```text
[0, 1]：合并 0 与 1 所在的组
[1, 2]：合并 1 与 2 所在的组
[3, 4]：合并 3 与 4 所在的组
```

处理完后：

```text
{0, 1, 2}
{3, 4}
```

最后判断 `source` 与 `destination` 是否有相同的代表节点（root）。相同表示在同一个连通分量中。

这个题只想判断或统计连通分量时，并查集很自然，因为它可以直接处理 `edges`，无需创建邻接表。

## 4. 并查集中的数据结构

```java
int[] parent;
int[] size;
```

### `parent[i]`

表示节点 `i` 当前指向的父节点。开始时每个节点各自是一组：

```text
parent = [0, 1, 2, 3, 4]
```

代表：

```text
{0} {1} {2} {3} {4}
```

### `find(node)`

寻找节点所属集合的代表节点（根节点）。两个节点根节点相同，说明它们属于同一个连通分量。

### `union(a, b)`

将 `a` 与 `b` 所在的两个集合合并。若它们原本已在同一组，就不必合并。

### `size[]`

记录集合大小。合并时让小集合挂到大集合下，可避免树变得太高。

## 5. 如果要统计连通分量数量

开始时：每个节点是独立分量，因此数量为 `n`。

每次 `union(a, b)` 成功把两个不同集合合并时：

```text
componentCount--
```

例如 `n = 5`、`edges = [[0,1], [3,4]]`：

```text
初始：{0} {1} {2} {3} {4}，共 5 组
合并 0、1：{0,1} {2} {3} {4}，共 4 组
合并 3、4：{0,1} {2} {3,4}，共 3 组
```

注意：孤立节点 `2` 也算一个连通分量。

## 6. 如果要返回每个连通分量的成员

并查集处理完所有边后：

1. 遍历每个节点；
2. 用 `find(node)` 找到其根节点；
3. 将根节点相同的节点收集到同一个 `List<Integer>`；
4. 最终使用 `List<List<Integer>>` 返回所有分量。

概念上的结果：

```text
[
  [0, 1, 2],
  [3, 4]
]
```

## 7. 复杂度

设边数为 `E`：

```text
时间复杂度：O(n + E × α(n))
空间复杂度：O(n)
```

`α(n)` 是一个增长极慢的函数，在实际算法题中几乎可以当作常数。因此并查集处理边的效率接近线性。

## 8. 这题应该记住什么？

```text
edges 是边列表，不是二维网格。
无向图中的“能否到达”，等价于“是否属于同一个连通分量”。
只需要处理边并合并集合时，可以直接用并查集，不必建邻接表。
需要从节点向邻居扩散、找路径、做网格题时，邻接表 + DFS/BFS 更自然。
```
