# Graph 已做题目总结：题目在问什么、为什么这样做

这份总结不以背代码为目标，而是把已经做过的题目放到同一张图算法地图里。每道题先说清楚“输入给了什么、要求返回什么”，再提炼真正需要学习的模型。

> 做图题的第一步不是选 DFS 还是 BFS，而是先翻译：节点是什么、边是什么、边有没有方向、题目最终要验证什么条件。

## 2. 这些题不是因为“长得像”才用同一种算法

### 可达性：1971、841

它们都在问“从起点出发能访问到什么”。

```text
1971：只要知道能否到达一个指定终点。
841：要知道是否能覆盖所有节点。
```

共同逻辑：从起点遍历所有可达节点，并用 `visited` 防止重复访问。

差别：1971 是无向图，一条边要双向加入邻接表；841 的钥匙只允许从当前房间指向能打开的房间，是有向关系。

### 对象图深拷贝：133

133 不只要求“访问过哪些节点”，还要求为每个原节点创建唯一的新节点，并恢复所有邻居关系。因此普通 `visited` 不够，需要保存：

```text
原节点 → 新节点
```

这个映射既防止图中的环造成无限递归，也让已经复制过的节点能够被其他复制节点正确引用。关键顺序是先创建并登记当前复制节点，再处理邻居。

### 连通分量：733、200、695

三题都把二维网格看成隐式图，但最终问题不同：

```text
733：只处理给定起点所在的一块区域。
200：扫描整个网格，统计一共有几块区域。
695：扫描整个网格，统计每块区域的大小并取最大值。
```

可迁移模板：

```text
外层扫描负责发现新的连通分量；
一次 DFS / BFS 负责完整处理当前连通分量；
在“发现新分量”或“访问分量节点”时更新题目要求的统计量。
```

### 层级扩散：994

994 不只是普通的“能否访问”。它要求最少分钟数，并强调所有腐烂橘子同时传播。

所以需要：

```text
所有初始腐烂橘子一起入队：多源；
固定每轮开始时的 queue.size()：分层；
一层代表一分钟：把 BFS 层数翻译成题目时间。
```

### 入度与依赖：1557、207

两题都需要理解入度，但对入度的使用不同：

```text
1557：只找原始入度为 0 的节点，不需要真的删除边。
207：不断删除入度为 0 的节点及其出边，观察能否删除所有节点。
```

这说明：看到同一个概念，不要立刻套同一段代码；先确定题目只是“读取性质”，还是需要“模拟状态变化”。

### 枚举路径：797

797 不是为了判断能否到达，而是必须返回所有合法路径。因此不能使用全局 `visited` 把一个节点永久封锁；同一个节点可能出现在多条不同路径中。

核心是：

```text
选择 next
→ 加入 path
→ 递归探索
→ 从 path 撤销 next
```

## 3. 一张选择算法的判断表

| 题目中的目标 | 优先想到 |
| --- | --- |
| 从一个点能否到另一个点 | DFS / BFS；无向连通性也可用并查集 |
| 找出起点所在的一整块区域 | 单次 DFS / BFS |
| 统计所有互不相连的区域 | 全局扫描 + 多次 DFS / BFS |
| 无权图最少几步 | 分层 BFS |
| 多个源同时传播最少几轮 | 多源、分层 BFS |
| 枚举所有路径 | DFS + 回溯 |
| 只判断无向图中是否同组 | 并查集 |
| 找有向依赖中的可执行顺序 / 环 | 拓扑排序 |

## 4. 建图时最容易混淆的四件事

### 边的方向

```text
无向边 [u, v]：u → v 和 v → u 都要记录。
有向边 [from, to]：只记录 from → to。
课程关系 [after, pre]：题目数组顺序是“课程、先修课”，建边却是 pre → after。
```

### 邻接表保存的是邻居，不是入度

```java
Map<Integer, List<Integer>> graph;
```

`graph.get(node).size()` 是从 `node` 出发的边数，也就是出度；不能拿它当其他节点的入度。

### 没有出边的节点可能不在 Map 中

如果只在看到 `from` 时创建列表，那么终点节点可能没有对应 key：

```java
graph.getOrDefault(node, Collections.emptyList())
```

这不是节点不存在，只是它没有后续邻居。

### 节点全集不能只从 Map 的 key 推断

题目给出 `numCourses = n` 时，节点全集是 `0` 到 `n - 1`。完全孤立的节点不会出现在任何边或邻接表中，但仍然必须计入答案。

## 5. 做完这些题真正应该获得的能力

1. 能把数组、房间、课程、网格翻译成“节点 + 边”。
2. 能区分无向边和有向依赖，知道邻接表该加一边还是两边。
3. 能区分“访问一个区域”“扫描所有区域”“按层计算最少轮数”。
4. 知道 `visited` 是图状态的一部分，不是 DFS 或 BFS 的专属写法。
5. 知道入度描述“还有多少前置条件未解决”，出度描述“解决当前节点会影响谁”。
6. 能用最终条件验证算法：访问目标、分量数量、剩余新鲜橘子、处理节点总数，而不是只看到队列为空就认为成功。

## 6. 每道图题开始前先写这几行

```text
节点：
边：
有向 / 无向：
显式图 / 隐式网格图：
起点有几个：
题目要求的最终结果：
完成后如何验证结果：
```

只要这七项说清楚，代码通常只是把模型翻译出来。
