# LeetCode 200：Number of Islands

> 难度：Medium  
> 主题：网格图（Grid Graph）、BFS / DFS、连通分量

## 1. 题目在问什么？

给定一个由 `'1'` 和 `'0'` 组成的二维网格：

```text
'1'：陆地
'0'：水
```

要求返回岛屿数量。

一座岛指的是：通过**上、下、左、右**连接在一起的一整块陆地。

注意：斜对角不算连接。

## 2. 为什么这是图题？

二维网格中的每个格子都可以看作一个节点：

```text
grid[row][col] = 一个节点
```

相邻关系由坐标隐含：

```text
上：(row - 1, col)
下：(row + 1, col)
左：(row, col - 1)
右：(row, col + 1)
```

因此这题本质上是在问：

> 网格中有多少个由 `'1'` 组成的连通分量？

## 3. BFS 和 DFS 都能做吗？

可以。

这题不要求最短路径，也不要求按层数扩散，只要求把同一座岛的所有陆地找出来并标记掉。

所以：

| 写法 | 是否可用 | 说明 |
| --- | --- | --- |
| BFS | 可以 | 用 `Queue` / `Deque` 一层层扩散 |
| DFS | 可以 | 用递归或栈一路走到底 |

Java 中如果网格很大，递归 DFS 可能有栈溢出风险；用 `Deque` 写 BFS 或迭代 DFS 更稳。

## 4. 本题需要的东西

```java
char[][] grid;          // 原始网格
boolean[][] visited;    // 是否已经处理过
Deque<int[]> queue;     // BFS 待处理坐标
int[][] directions;     // 四个方向
int res;                // 岛屿数量
```

你的当前写法使用的是 BFS：

```text
queue 负责保存接下来要扩散的陆地坐标。
visited 负责防止同一块陆地被重复处理。
directions 负责统一处理上下左右四个方向。
res 负责统计发现了几座新岛。
```

## 5. 外层扫描与 BFS 的关系

这题和 Flood Fill 最大的区别是：

```text
Flood Fill：只处理一个给定起点所在的区域。
Number of Islands：要扫描完整地图，统计所有区域。
```

因此需要两层逻辑：

```text
外层扫描：负责发现新岛的起点。
BFS / DFS：负责把这一整座岛全部标记完。
```

当扫描到一个格子时：

```text
如果已经访问过：跳过。
如果是水：跳过。
如果是未访问过的陆地：说明发现一座新岛。
```

发现新岛后，从这个格子开始做一次完整 BFS / DFS，把同一座岛上的陆地全部标记为已访问。

## 6. 计数时机

每发现一个“未访问过的陆地”，就代表发现了一座新岛。

因此岛屿数量只应该在这个时机增加一次：

```text
扫描到未访问的 '1'
→ 启动一次 BFS / DFS
→ 这一整块陆地都属于同一座岛
→ 岛屿数量加一
```

不要在 BFS 遍历到每个陆地格子时都加一，否则统计的是陆地格子数量，不是岛屿数量。

## 7. 四方向数组

本题只允许上下左右连接：

```java
int[][] directions = {
    {-1, 0}, // 上
    {1, 0},  // 下
    {0, -1}, // 左
    {0, 1}   // 右
};
```

不包含：

```text
左上、右上、左下、右下
```

所以斜着相邻的两个 `'1'` 不是同一座岛。

## 8. 邻居检查顺序

从当前格子扩散到邻居时，按这个顺序判断：

```text
1. 是否越界？
2. 是否已经访问过？
3. 是否是水？
4. 如果是未访问过的陆地：标记，并加入待处理结构。
```

重点习惯：

> 入队时就标记为已访问。

这样可以避免同一个格子从多个方向被重复加入队列。

## 9. 当前代码的结构判断

你的当前代码整体结构是 BFS 版本：

```text
1. 先创建 visited。
2. 用双层循环扫描完整 grid。
3. 遇到水或访问过的格子就跳过。
4. 遇到未访问过的陆地，就从这里启动 BFS。
5. BFS 内部用四方向扩散，把同一座岛全部标记。
6. BFS 结束后，岛屿数量增加一次。
```

这个结构是对的。

## 10. 复杂度

设网格有 `m` 行、`n` 列。

```text
时间复杂度：O(m × n)
空间复杂度：O(m × n)
```

原因：

```text
每个格子最多被访问一次。
每个格子最多检查四个方向。
visited 和 queue 最坏情况下都可能占用 O(m × n) 空间。
```

## 11. 本题检查清单

```text
是否扫描了完整地图？
是否只在发现新岛起点时计数？
是否只允许上下左右连接？
是否跳过了水？
是否跳过了已访问格子？
是否在入队时就标记 visited？
```
