# Lecture 1：Graph（图）基础

## 图是什么？

图（Graph）是一种表示“对象之间关系”的数据结构。

生活中的许多问题都不是线性的：一个人可以认识很多人；一个城市可以通往多个城市；一门课可能依赖多门前置课程。只要问题的重点是“谁和谁相连、能不能到达、先后依赖如何安排”，就很可能可以用图表示。

```text
A —— B —— C
|    |
D    E
```

上图表示五个对象之间的连接关系。

## 基本组成

### 顶点（Vertex / Node）

图中的每一个对象叫作**顶点**，也常叫**节点**。

在不同问题中，顶点可以是：

- 人：社交关系图
- 城市：地图与路线问题
- 网页：网页链接关系
- 课程：课程依赖问题
- 二维数组中的一个格子：迷宫、岛屿问题

### 边（Edge）

两个顶点之间的关系叫作**边**。

例如“北京和上海之间有一条航线”，北京、上海是顶点，航线就是边。

## 图论题的第一步：建模

拿到一道题时，先不要急着想算法。先回答下面四个问题：

```text
1. 节点是什么？
2. 边是什么？
3. 边有方向吗？
4. 边有权重吗？
```

例如城市路线题：城市是节点、道路是边；单行道是有向边；距离或车费是边的权重。

## 常见分类

### 无向图（Undirected Graph）

连接没有方向，`A — B` 表示 A 能到 B，B 也能到 A。

例如：好友关系、双向道路。

```text
A —— B
```

### 有向图（Directed Graph）

连接有方向，`A → B` 只表示能从 A 到 B，不一定能从 B 回到 A。

例如：关注关系、课程前置关系、单行道。

```text
A → B
```

### 有权图（Weighted Graph）

边上有额外数值，称为**权重**；它可以表示距离、时间、费用或风险等。

```text
A --5--> B
```

这里的 `5` 可以表示从 A 到 B 的距离是 5 公里。

### 无权图（Unweighted Graph）

边只表示“有连接”或“没连接”，不关心距离和费用。

## 图的两种常见存储方式

### 邻接表（Adjacency List）

记录每个节点直接连接了哪些节点。

```text
A: B, D
B: A, C, E
C: B
D: A
E: B
```

适合“实际连接关系不多”的图，也是算法题中最常见的表示方式。

### 邻接矩阵（Adjacency Matrix）

用一个二维表记录任意两点之间是否有边：有边写 `1`，没有写 `0`。

```text
    A B C
A   0 1 0
B   1 0 1
C   0 1 0
```

它查询两点是否直接相连很快，但会占用较多空间。

## 图中的重要概念

### 度（Degree）

一个节点直接连接的边数，叫作这个节点的**度**。

```text
    A
    |
B — C — D
    |
    E
```

节点 `C` 连着四条边，因此它的度是 4。

在有向图中还要区分：

- **入度（in-degree）**：有多少条边指向该节点；
- **出度（out-degree）**：有多少条边从该节点指出去。

### 路径（Path）

从一个节点经过若干条边走到另一个节点的路线。

例如 `A → B → C` 是从 A 到 C 的一条路径。

### 连通（Connected）

如果从 A 能沿着边走到 B，就说 A 和 B 连通。

### 连通分量（Connected Component）

在无向图中，一组内部任意两点都能互相到达、且无法连接到其他节点的点，叫一个连通分量。

```text
A —— B      C —— D
```

这张图有两个连通分量：`{A, B}` 与 `{C, D}`。

### 环（Cycle）

沿着边走一圈又回到起点，称为环。

```text
A —— B
|    |
D —— C
```

`A → B → C → D → A` 就是一个环。

## 图题通常在问什么？

看到以下关键词时，可以考虑图：

- 是否能从 A 到 B？—— 可达性 / 路径问题
- 最少经过几步、最短距离是多少？—— 最短路径问题
- 有多少组互不相连的对象？—— 连通分量问题
- 是否存在循环依赖？—— 环检测
- 有依赖关系的任务如何排序？—— 拓扑排序
- 哪些节点属于同一个集合？—— 并查集

## 常见遍历方法

### DFS：深度优先搜索

从一个节点出发，先沿着一条连接不断向前走；走不下去后再回退，尝试其他方向。常用于找连通区域、检测环、回溯等。

### BFS：广度优先搜索

从一个节点出发，先访问所有距离为 1 的邻居，再访问距离为 2 的节点，一层层向外扩展。无权图中常用于求“最少走几步”。

## 二维数组为什么常被看作图？

网格中的每一个格子都可以作为一个顶点；相邻格子之间可以看作有边。因此迷宫、感染扩散、岛屿、矩阵中的区域等题，通常都是**隐式图**：题目没有把点和边画出来，但它们已经藏在网格的相邻关系中。

## 总结

> 图不是某一道题的专属算法，而是一种抽象关系模型：顶点表示对象，边表示对象之间的联系。

学习图时，先判断节点是什么、边代表什么、边有无方向和权重，再判断题目是在问连通、路径、最短距离、环，还是依赖顺序。
