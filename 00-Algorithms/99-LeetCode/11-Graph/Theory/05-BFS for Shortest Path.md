# Lecture 6：BFS 与无权图最短路径

> 前置：[[02-Graph Traversal - DFS & BFS]]、[[04-Grid Graph & Connected Components]]

## 1. “最短路径”到底是什么意思？

如果图的每条边都没有不同的代价，或每走一步的代价都一样为 1，那么最短路径指的是：

> 从起点到终点，最少需要经过多少条边 / 多少步。

例如：

```text
0 —— 1 —— 3
 \       /
  2 ————
```

从 `0` 到 `3`：

```text
0 → 1 → 3：2 步
0 → 2 → 3：2 步
```

最短距离为 2。

## 2. 为什么 DFS 不保证最短？

DFS 会先沿一条路尽量走深。若它先走到一条很长的路线，可能在看到更短路线之前已经到达终点。

```text
0 —— 1 —— 2 —— 3 —— 4
 \___________________/
```

从 `0` 到 `4` 有一条直接边，也有一条很长的路线。DFS 访问邻居的顺序不同，可能先沿长路线走；它能判断“能否到达”，但第一次到达终点不一定是最短路线。

## 3. BFS 为什么保证最短？

BFS 使用队列，严格按照“离起点的步数”一层层向外访问：

```text
第 0 层：起点
第 1 层：起点一步能到的所有节点
第 2 层：起点两步能到的所有节点
第 3 层：起点三步能到的所有节点
```

因此，某个节点第一次被 BFS 访问到时，使用的步数必然最少。因为所有更少步数的节点都已经在之前的层处理完了。

## 4. 队列中的“层”是如何形成的？

设起点为 `0`：

```text
0 的邻居：1、2
1 的邻居：3、4
2 的邻居：5
```

队列过程大致为：

```text
初始：          [0]
处理 0 后：      [1, 2]
处理 1 后：      [2, 3, 4]
处理 2 后：      [3, 4, 5]
```

`1、2` 都是第 1 层；`3、4、5` 都是第 2 层。因为先进入队列的第 1 层节点会先被取出，后续加入的第 2 层节点自然排在后面。

## 5. 两种记录距离的方式

### 方法 A：单独使用 `distance[]`

```text
distance[start] = 0
当从 current 走到 next：
distance[next] = distance[current] + 1
```

适合：希望知道每个节点距离起点多远。

### 方法 B：按队列层数处理

每一轮先保存当前队列长度 `levelSize`，只处理这一批节点；处理完一批后，步数加一。

```text
steps = 0
队列不为空时：
    本层有 levelSize 个节点
    连续取出 levelSize 个节点并扩展邻居
    steps 加一
```

适合：只关心到达终点需要多少步，或题目使用“第几分钟 / 第几轮扩散”的描述。

## 6. `visited` 与最短路径的关系

无权图 BFS 中，节点**第一次入队**时就应该标记 `visited`。这是因为第一次到达它的路径一定最短。

若延迟到出队才标记，同一个节点可能从多个邻居被重复放入队列，造成额外工作，也让距离处理变得混乱。

## 7. 哪些题可以直接用 BFS 最短路？

题目中出现下列词语时，应考虑 BFS：

```text
最少步数
最少移动次数
最少转换次数
最近的
第几分钟扩散
每一步代价相同
```

例如：迷宫最少步数、单词接龙、最少操作次数、腐烂橘子的最短时间。

## 8. 重要限制：边权必须相同

BFS 的最短路结论依赖于：

```text
每走一条边的代价都相同
```

如果某条路耗时 1 分钟、另一条路耗时 10 分钟，边权不同，BFS 只会计算经过的边数，不能保证总耗时最少。此时需要学习 Dijkstra。

## 9. 复杂度

邻接表图中，BFS 最多访问每个节点一次、检查每条边有限次：

```text
时间复杂度：O(V + E)
空间复杂度：O(V)
```

网格图中：

```text
时间复杂度：O(m × n)
空间复杂度：O(m × n)
```
