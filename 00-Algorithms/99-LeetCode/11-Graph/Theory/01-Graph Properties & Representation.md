# Lecture 2：图的性质与表示方式

> 前置：[[00-Graph Basics]]

## 1. 路径与可达性

**路径（path）**是从一个节点沿着一条或多条边走到另一个节点的路线。

```text
0 —— 1 —— 2 —— 3
```

从 `0` 到 `3` 的路径是 `0 → 1 → 2 → 3`。只要存在至少一条路径，就说 `0` 可以到达 `3`，或两者**连通**。

LeetCode 1971 问的就是：`source` 是否能到达 `destination`。

## 2. 连通分量（Connected Component）

无向图中，内部的节点可以相互到达、但与其他区域完全没有边连接的一整块区域，叫一个**连通分量**。

```text
0 —— 1 —— 2      3 —— 4
```

这张图有两个连通分量：`{0, 1, 2}` 与 `{3, 4}`。

以下问题本质上都是连通分量问题：

- 有多少个朋友圈？
- 有多少个省份？
- 有多少座岛屿？
- A 和 B 是否属于同一组？

## 3. 环（Cycle）

如果从一个节点沿边走一圈后回到原点，就形成一个**环**。

```text
0 —— 1
|    |
3 —— 2
```

`0 → 1 → 2 → 3 → 0` 是一个环。

环在无向图（道路、好友）中很正常；但在课程依赖等有向图中，环常意味着互相依赖、无法安排先后顺序。

## 4. 有向图中的入度、出度与 DAG

对于有向边 `A → B`：

```text
A 增加一条出边，因此 A 的出度加一；
B 增加一条入边，因此 B 的入度加一。
```

在依赖关系中，可以把入度理解成“当前节点还有多少个直接前置条件”。入度为 `0` 的节点没有等待中的前置条件，可以作为当前的处理起点。

**有向无环图（Directed Acyclic Graph，DAG）** 是没有有向环的有向图。课程安排、任务流水线、构建依赖等问题，经常要求判断依赖图是否为 DAG，或给出一个满足依赖的拓扑顺序。

```text
有向图不一定是 DAG；
只有不存在有向环时，才是 DAG；
DAG 至少存在一个拓扑序。
```

详见 [[07-Topological Sort]]。

## 5. 稀疏图与稠密图

- **稀疏图**：节点很多，但实际连接很少。例如社交网络中，你不会认识所有人。
- **稠密图**：大量节点两两之间都可能连接。

这一区别会影响存图方式：算法题多数是稀疏图，因此常用邻接表。

## 6. 三种常见表示方式

### 边列表（Edge List）

```text
edges = [[0,1], [1,2], [2,3]]
```

每个二元数组表示一条边，图为：

```text
0 —— 1 —— 2 —— 3
```

优点是输入直观；缺点是要找某节点的所有邻居时，通常得扫描所有边。

### 邻接表（Adjacency List）

```text
0: [1]
1: [0, 2]
2: [1, 3]
3: [2]
```

每个节点保存所有直接邻居。DFS/BFS 最常用这种形式，因为到达一个节点后，可以立刻知道下一步能去哪里。

### 邻接矩阵（Adjacency Matrix）

```text
    0 1 2
0   0 1 0
1   1 0 1
2   0 1 0
```

第 `i` 行第 `j` 列表示 `i` 与 `j` 是否直接相连。

- 优点：判断两点是否直接相连很快。
- 缺点：需要 `n × n` 空间；节点多、边少时浪费空间。

## 7. 网格图（隐式图）

```text
1 1 0
0 1 0
0 0 1
```

网格题没有显式写出边，但可以把每一个格子看成节点；通常上、下、左、右相邻的格子之间存在边。岛屿、迷宫、Flood Fill、感染扩散都属于网格图。
