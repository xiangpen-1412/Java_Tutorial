# Lecture 8：拓扑排序（Topological Sort）与有向图环检测

## 1. 拓扑排序解决什么问题？

拓扑排序处理的是有向图中的依赖先后关系。

例如：

```text
先学 Java 基础 → 再学集合框架 → 再学并发
先完成需求分析 → 再写代码 → 再测试
```

如果一条有向边是：

```text
A → B
```

在依赖模型中通常表示 `A` 必须在 `B` 之前完成。

拓扑序就是一个节点排列，使每条边的起点都出现在终点之前。

## 2. 什么图才有拓扑序？

只有有向无环图（Directed Acyclic Graph，DAG）存在拓扑序。

如果存在：

```text
A → B → C → A
```

就会得到互相矛盾的要求：

```text
A 要在 B 前
B 要在 C 前
C 又要在 A 前
```

没有任何节点可以作为真正起点，所以不存在合法顺序。

注意：拓扑序通常不唯一。只要所有依赖边的先后条件都满足，不同顺序都可能正确。

## 3. 入度和出度的含义

对于有向边：

```text
A → B
```

- 它增加 `A` 的出度；
- 它增加 `B` 的入度。

在依赖问题中，可以把入度理解为：

> 当前节点还有多少个直接前置条件没有被解决。

入度为 `0` 的节点没有等待中的前置条件，所以可以立即执行。

邻接表通常保存出边：

```text
graph[A] = 所有被 A 影响的后续节点
```

入度数组或 Map 单独保存每个节点还在等待多少条入边。

## 4. Kahn 算法：基于入度的 BFS

最常用的拓扑排序方法是 Kahn 算法。

```text
1. 统计每个节点的入度。
2. 把所有入度为 0 的节点放入队列。
3. 从队列取出一个节点，加入拓扑序或计为已处理。
4. 删除该节点的所有出边：让每个后继节点入度减一。
5. 后继节点入度变成 0 时，将它加入队列。
6. 重复直到队列为空。
```

所谓“删除边”，通常不是真的修改邻接表，而是把后继节点的入度减一。

## 5. 为什么它一定能得到合法顺序？

每次只选择入度为 `0` 的节点，因此它当前没有任何未处理的前驱。

把它处理后再解除它对后继的限制，可以保证：

```text
一个节点进入结果之前，它所有仍存在的前置约束都已经消失。
```

所以取出节点的顺序天然满足所有依赖关系。

## 6. 为什么可以用它检测环？

如果图中没有环，总能不断找到至少一个入度为 `0` 的节点，最终处理完全部节点。

如果存在环：

```text
A → B → C → A
```

环内每个节点都至少有一条来自环内的入边。即使处理完环外节点，环内节点的入度仍不会变成 `0`，所以它们永远不能进入队列。

最终：

```text
processedCount == nodeCount：无环，存在拓扑序
processedCount < nodeCount：有环，不存在拓扑序
```

## 7. 一次完整过程

```text
0 → 1
0 → 2
1 → 3
2 → 3
```

初始入度：

```text
0: 0
1: 1
2: 1
3: 2
```

执行：

```text
队列：[0]
取出 0 → 1、2 入度变为 0 → 队列 [1,2]
取出 1 → 3 入度变为 1
取出 2 → 3 入度变为 0 → 队列 [3]
取出 3 → 完成
```

一种拓扑序是 `[0, 1, 2, 3]`。如果先处理 `2`，`[0, 2, 1, 3]` 也合法。

## 8. 通用 Java 模板

节点编号为 `0` 到 `n - 1` 时，数组通常比 Map 更直接：

```java
List<List<Integer>> graph = new ArrayList<>();
for (int node = 0; node < n; node++) {
    graph.add(new ArrayList<>());
}

int[] indegree = new int[n];

for (int[] edge : edges) {
    int from = edge[0];
    int to = edge[1];

    graph.get(from).add(to);
    indegree[to]++;
}

Deque<Integer> queue = new ArrayDeque<>();
for (int node = 0; node < n; node++) {
    if (indegree[node] == 0) {
        queue.offer(node);
    }
}

int processedCount = 0;

while (!queue.isEmpty()) {
    int current = queue.poll();
    processedCount++;

    for (int next : graph.get(current)) {
        indegree[next]--;

        if (indegree[next] == 0) {
            queue.offer(next);
        }
    }
}

boolean hasTopologicalOrder = processedCount == n;
```

如果需要返回具体顺序，就在节点出队时把它加入 `order`。如果只判断是否可行，只维护 `processedCount` 即可。

## 9. BFS 拓扑排序和普通 BFS 的区别

普通 BFS 通常从一个指定起点出发，按距离一层层访问邻居。

拓扑 BFS：

```text
起点不是题目指定的一个节点，而是所有入度为 0 的节点；
入队条件不是“第一次访问”，而是“所有前置条件都已解决”；
它的层数通常不表示最短距离，而表示依赖可以被逐轮解除。
```

因此，拓扑排序虽然使用队列，却不等同于无权最短路径 BFS。

## 10. DFS 也可以做拓扑排序吗？

可以。DFS 使用三种状态检测有向环：

```text
0：尚未访问
1：正在当前递归路径中访问
2：已经完整处理
```

沿边访问时，如果遇到状态为 `1` 的节点，说明回到了当前路径中的祖先，存在有向环。

如果没有环，节点在 DFS 完成时加入结果，最后把结果反转，就能得到拓扑序。

初学时建议先掌握入度 BFS，因为“前置条件数量减少”的含义更直观，也更贴合 Course Schedule。

## 11. 最常见错误

### 把边方向建反

先用一句自然语言确定：谁必须先完成，谁是后续。再建：

```text
前置节点 → 后续节点
```

### 把出度当入度

邻接表列表长度通常是出度，不能用来寻找入度为 `0` 的节点。

### 漏掉没有出现在边中的节点

节点全集来自 `n`。孤立节点入度为 `0`，也属于合法拓扑序。

### 入度不是简单设成 0

处理一个前驱只删除一条入边，因此必须每次减一；只有所有前驱都处理完成后才会变成 `0`。

### 同一个节点重复入队

只有入度从 `1` 变成 `0` 的那一刻入队。不要使用 `<= 0`，也不要重复减入度。

### 只判断队列是否为空

队列最终一定会为空。真正的成功条件是处理数量等于节点总数。

## 13. 复杂度

设节点数为 `V`、边数为 `E`：

```text
时间复杂度：O(V + E)
空间复杂度：O(V + E)
```

每个节点最多入队一次，每条边只被处理一次。

## 14. 本讲检查清单

```text
题目描述的是有向依赖吗？
一条输入关系应该建成谁 → 谁？
是否初始化了全部节点？
入度增加的是边的终点吗？
初始队列是否包含所有入度为 0 的节点？
处理当前节点时，是否只把每条出边对应的入度减一次？
最终是否用 processedCount == nodeCount 验证无环？
```

## 15. 一句话总结

> 拓扑排序不断选择当前没有未解决前置条件的节点，并解除它对后续节点的限制；如果这个过程无法处理全部节点，剩余节点中就存在循环依赖。
