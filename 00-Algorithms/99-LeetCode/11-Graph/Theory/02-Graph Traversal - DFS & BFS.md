# Lecture 3：图遍历——DFS 与 BFS

> 前置：[[00-Graph Basics]]、[[01-Graph Properties & Representation]]

## 1. 什么是遍历？

图遍历就是：从一个节点出发，按照某种规则访问所有能到达的节点。

例如：从城市 `0` 出发，看看能否到达城市 `5`；或者从一块陆地出发，找出同一座岛的所有陆地。

## 2. 为什么必须记录 visited？

图可能有环：

```text
0 —— 1
|    |
2 —— 3
```

若不记住已经访问的节点，可能在 `0 → 1 → 0 → 1` 之间无限往返。

`visited`（已访问记录）有三个作用：

- 防止在环里重复走；
- 防止同一节点被反复处理；
- 使总工作量保持在每个节点、每条边至多处理有限次。

## 3. DFS：深度优先搜索

DFS 的直觉是：选一条路后先尽量向深处走，走不通再回退，尝试其他路。

```text
0 —— 1 —— 3
     |
     2
```

从 `0` 出发，可能先走 `0 → 1 → 3`；到头后回到 `1`，再走向 `2`。

常见实现方式：递归或显式栈。

适用场景：

- 判断两点是否连通；
- 找出一个完整连通区域；
- 统计连通分量 / 岛屿；
- 检测环；
- 回溯搜索。

## 4. BFS：广度优先搜索

BFS 的直觉是：从起点开始，先访问所有“一步能到”的节点，再访问“两步能到”的节点，像水波一样一层层扩散。

```text
起点：0
第 1 层：1、2
第 2 层：3、4、5
```

常见实现方式：队列。

适用场景：

- 判断连通性；
- 无权图中求最少经过几条边；
- 最少操作次数；
- 按时间扩散的问题，如腐烂橘子。

## 5. DFS 和 BFS 怎么选？

| 题目目标 | 常用选择 |
| --- | --- |
| 只问能否到达 / 统计区域 | DFS 或 BFS 都行 |
| 无权图的最少步数 | BFS |
| 逐分钟、逐轮扩散 | BFS |
| 需要探索所有分支、检测环 | DFS 常更自然 |

对 LeetCode 1971 而言，DFS 与 BFS 都可以；题目只关心能否从起点遍历到终点。

## 6. 遍历复杂度的直觉

设节点数为 `V`、边数为 `E`。使用邻接表时，DFS/BFS 通常为：

```text
时间：O(V + E)
空间：O(V)
```

因为每个节点会标记一次，每条边也只会被检查有限次。
