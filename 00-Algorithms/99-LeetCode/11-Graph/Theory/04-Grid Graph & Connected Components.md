# Lecture 5：网格图与连通分量

> 前置：[[02-Graph Traversal - DFS & BFS]]、[[03-Graph Algorithms Map]]

## 1. 为什么二维数组也是图？

先看一个二维数组：

```text
1 1 0
1 0 0
0 0 1
```

它表面上只是一个表格，但我们可以规定：**每个格子都是一个节点**。

```text
第 r 行、第 c 列的格子 = 节点 (r, c)
```

若两个格子上下或左右相邻，就认为它们之间有一条边：

```text
    (r - 1, c)  上
(r, c - 1)  左  (r, c)  当前  (r, c + 1)  右
    (r + 1, c)  下
```

因此，岛屿、迷宫、图片填色、感染扩散这类二维数组题，本质上都是图题。它们的边没有写成 `edges = [[0, 1], ...]`，而是藏在“上下左右相邻”这一规则中，这种图叫做**隐式图**。

## 2. 不需要建立邻接表

普通图常通过邻接表记录“节点 x 的邻居是谁”。网格图不用这么做，因为当前格子 `(r, c)` 的邻居可以直接算出：

```text
上：(r - 1, c)
下：(r + 1, c)
左：(r, c - 1)
右：(r, c + 1)
```

代码中常将方向统一保存：

```java
int[][] directions = {
    {-1, 0}, // 上
    {1, 0},  // 下
    {0, -1}, // 左
    {0, 1}   // 右
};
```

对于当前坐标 `(row, col)`，某一方向的邻居为：

```java
int nextRow = row + direction[0];
int nextCol = col + direction[1];
```

每个邻居都要先检查是否越界。

## 3. 什么是网格中的连通分量？

在下面的网格中，假设只有 `1` 可以通行：

```text
1 1 0 0
1 0 0 1
0 0 1 1
```

按上下左右连通规则，存在两块 `1` 区域：

```text
{(0,0), (0,1), (1,0)}
{(1,3), (2,2), (2,3)}
```

每一块就是一个连通分量。题目可能要求：

| 题目问法 | 实际要做的事 |
| --- | --- |
| 从某个起点能走到哪里？ | 找起点所在的一个连通分量 |
| 有几座岛？ | 统计所有连通分量的数量 |
| 最大岛屿面积？ | 求每个连通分量的大小，再取最大值 |
| 把同色区域改色？ | 修改起点所在连通分量 |

## 4. 网格遍历的通用检查顺序

准备访问一个邻居 `(nextRow, nextCol)` 时，按这个顺序判断：

```text
1. 坐标是否越界？
2. 是否已访问？
3. 这个格子是否满足题目定义的“可走条件”？
4. 若可以：立即标记，再加入 DFS/BFS 的待处理结构。
```

“可走条件”由题目决定：

```text
Flood Fill：颜色是否等于起点原始颜色
岛屿题：格子是否为陆地
迷宫题：格子是否不是墙
感染题：格子是否是可被感染的对象
```

## 5. 只处理一个区域：Flood Fill 型

如果题目给了一个起点，只需把起点放入待处理结构，然后不断扩散：

```text
将起点标记为已访问
起点入队（BFS）或入栈（DFS）

只要待处理结构不为空：
    取出一个格子
    对四个方向逐个检查
    合格邻居：标记为已访问，并加入待处理结构
```

最终被访问到的所有格子，就是起点所在的连通分量。

## 6. 统计所有区域：岛屿型

若题目问“有几块区域”，只从一个起点出发不够。必须扫描整个网格：

```text
componentCount = 0

遍历每个格子：
    如果它是有效格子，并且尚未访问：
        componentCount 加一
        从该格子进行一次完整 DFS/BFS
```

为什么每发现一个未访问的有效格子，数量就加一？

因为此前的遍历已经完整处理过所有旧区域；现在仍遇到一个未访问的有效格子，说明它不属于任何已知区域，只能是一个新连通分量的开始。

## 7. `visited` 的两种写法

### 单独记录访问状态

```java
boolean[][] visited = new boolean[rows][cols];
```

优点：不修改原始网格，逻辑清楚。

### 直接修改网格

例如将已处理的陆地从 `'1'` 改为 `'0'`，或将旧颜色改成新颜色。

优点：节省 `visited` 的额外空间。

注意：如果题目还需要依靠原始值作判断，必须在修改前保存必要信息，例如 Flood Fill 中的 `originalColor`。

## 8. BFS 与 DFS 在网格题中的选择

| 目标 | 推荐 |
| --- | --- |
| 只需要找到 / 统计一个区域 | DFS 或 BFS 都可 |
| 求区域大小 | DFS 或 BFS 都可 |
| 求最少移动步数 | BFS |
| 按分钟 / 按轮次扩散 | BFS |

网格很大时，Java 的递归 DFS 可能因为递归太深而栈溢出；迭代 DFS（`Deque` 当栈）或 BFS（`Queue`）更稳妥。

## 9. 复杂度

设网格有 `m` 行、`n` 列。一个正确的遍历会让每个格子最多访问一次，并对每个格子检查至多四个方向：

```text
时间复杂度：O(m × n)
空间复杂度：O(m × n)
```
