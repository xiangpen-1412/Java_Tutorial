# Lecture 7：并查集（Union-Find）与连通分量

> 前置：[[01-Graph Properties & Representation]]、[[03-Graph Algorithms Map]]

## 1. 并查集解决什么问题？

并查集（Union-Find / Disjoint Set Union，DSU）用来维护“谁和谁属于同一组”。

典型问题：

```text
给出很多无向边后：
两个节点是否连通？
图中还剩多少个连通分量？
每个连通分量有哪些节点？
```

例如：

```text
边：[0,1]、[1,2]、[3,4]
```

最终分组为：

```text
{0, 1, 2}
{3, 4}
```

## 2. 与 DFS / BFS 的区别

DFS / BFS 的思路是：从一个节点出发，沿着邻居一步步走。

并查集的思路是：每读到一条边，就把边两端的两个集合合并。

```text
读到 [0, 1]：合并 {0} 与 {1}
读到 [1, 2]：合并 {0,1} 与 {2}
读到 [3, 4]：合并 {3} 与 {4}
```

并查集不需要邻接表，适合输入本身就是边列表、且主要问题是“是否同组 / 连通分量”的场景。

## 3. 核心数据结构：`parent[]`

```java
int[] parent = new int[n];
```

开始时每个节点各自是一组：

```text
parent = [0, 1, 2, 3, 4]
```

意思是每个节点都指向自己，是自己这一组的代表：

```text
{0} {1} {2} {3} {4}
```

并查集把每个集合表示成一棵树。树根叫做该集合的**代表节点（root）**。

## 4. `find`：找到一个节点所在集合的根

如果节点不断沿 `parent` 向上走，最终会走到一个满足以下条件的节点：

```text
parent[root] == root
```

这个 `root` 就是节点所属集合的代表。

```text
2 → 1 → 0
parent[0] = 0
```

因此 `find(2)` 的结果是 `0`。若两个节点的根相同，它们就在同一个连通分量中。

## 5. `union`：合并两个集合

处理一条边 `[a, b]` 时：

```text
rootA = find(a)
rootB = find(b)
```

- 若 `rootA == rootB`：两点本来就在同一组，无需操作。
- 若不同：让其中一个根指向另一个根，两组就合并了。

```text
{0,1} 与 {2}
将 root(2) 指向 root(0)
结果：{0,1,2}
```

## 6. 两个关键优化

### 路径压缩（Path Compression）

当 `find(2)` 发现路径是 `2 → 1 → 0` 后，可以直接让 `2` 指向 `0`：

```text
原来：2 → 1 → 0
压缩：2 ─────→ 0
```

之后再找 `2` 的根会更快。

### 按大小合并（Union by Size）

维护 `size[root]`，合并时将小树接到大树下面，避免树过高。

```text
小集合 → 大集合
```

配合路径压缩后，实际运行几乎可以看作每次操作都是常数时间。

## 7. 如何统计连通分量数量？

开始时 `n` 个节点彼此独立，因此：

```text
componentCount = n
```

每次 `union` 成功合并两个原本不同的集合时：

```text
componentCount--
```

如果一条边两端本来就在同一组，不能减少数量。

## 8. 如何返回每个连通分量？

处理完全部边后：

```text
遍历每个节点 node：
    root = find(node)
    将 node 放入以 root 为 key 的列表中
```

可以使用：

```java
Map<Integer, List<Integer>> groups;
```

最终把 `groups.values()` 转成：

```java
List<List<Integer>>
```

## 9. 并查集的适用边界

并查集适合：

- 判断无向图中的连通性；
- 统计连通分量；
- 不断加入边后判断是否连通；
- 检测无向图加边时是否成环；
- Kruskal 最小生成树算法。

并查集不适合：

- 输出从 A 到 B 的具体路径；
- 求最短路径；
- 处理有方向的可达性关系。

这些场景通常应使用 DFS、BFS、Dijkstra 或拓扑排序。

## 10. 复杂度

设边数为 `E`：

```text
初始化：O(n)
处理全部边：O(E × α(n))
空间：O(n)
```

`α(n)` 增长极慢，在实际数据范围内可近似看作常数，因此并查集处理边的总耗时接近线性。

## 本讲检查清单

```text
输入是不是边列表？
题目是否只关心“是否连通 / 是否同组 / 有几组”？
是否不需要具体路径？
若都是“是”，并查集通常是一个好选择。
```
