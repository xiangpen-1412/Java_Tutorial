# 235. Lowest Common Ancestor of a Binary Search Tree

> 难度：Medium  
> 主题：Binary Search Tree、最近公共祖先、递归、区间判断

## 问题

给定一棵二叉搜索树以及树中的两个节点 `p` 和 `q`，寻找它们的最近公共祖先。公共祖先必须同时是两个节点的祖先；题目允许一个节点成为自己的祖先，因此当 `p` 是 `q` 的祖先时，答案可以是 `p` 本身，反之亦然。

`p` 和 `q` 只是参数名称，题目没有规定 `p.val < q.val`，所以应先整理出两个值的上下界：

```java
int smaller = Math.min(p.val, q.val);
int larger = Math.max(p.val, q.val);
```

## 核心收获

- **识别信号**：题目要求在 BST 中同时定位两个节点。BST 的左子树都小于当前值、右子树都大于当前值，因此无需分别搜索两条完整路径。
- **核心转换**：把 `p.val` 和 `q.val` 看作区间 `[smaller, larger]`，观察当前 `root.val` 位于区间左侧、右侧还是区间内部。
- **递归契约**：`lowestCommonAncestor(root, p, q)` 表示“在以 `root` 为根的 BST 中，返回 `p` 和 `q` 的最近公共祖先”。每次利用大小关系排除一整侧子树。

## 三种位置关系

```text
root.val < smaller
→ 两个目标值都比 root 大，只可能继续出现在右子树

root.val > larger
→ 两个目标值都比 root 小，只可能继续出现在左子树

smaller <= root.val <= larger
→ 两个目标从这里开始分处两侧，或 root 本身就是其中一个目标
```

关键在于比较的是当前节点 `root.val` 与两个目标值的上下界，而不是比较 `root.left.val` 或 `root.right.val`。

## 结构骨架

```java
int smaller = Math.min(p.val, q.val);
int larger = Math.max(p.val, q.val);

if (root.val < smaller) {
    // 处理右子树
} else if (root.val > larger) {
    // 处理左子树
} else {
    // 当前节点是分叉点，或等于其中一个目标
}
```

## 为什么区间判断成立

当当前值小于两个目标值时，根据 BST 性质，两个目标都不可能在当前节点的左子树；大于两个目标值时同理。只有当前值落在两个目标值之间时，两个目标才不会继续同时落在同一侧，因此当前节点是它们路径分开的最低位置。

若当前节点恰好等于 `p` 或 `q`，它也落在闭区间 `[smaller, larger]` 中。结合“节点可以是自己的祖先”这一条件，不需要另写相等分支。

## 复杂度

- 时间复杂度：`O(h)`，其中 `h` 是树高；每次只进入一侧子树。
- 递归栈空间：`O(h)`。
- 平衡 BST 中 `h = O(log n)`；树退化成链表时 `h = O(n)`。

## 变种与注意点

- 该方法依赖 BST 的大小关系；如果是普通二叉树，不能根据节点值排除一整侧子树。
- 题目通常保证 `p` 和 `q` 都存在于树中。若没有这一保证，仅凭分叉位置不能证明两个目标实际存在，还需要额外验证。
- 直接访问 `root.left.val` 或 `root.right.val` 不仅判断对象不合适，孩子为空时还可能触发 `NullPointerException`。
- 若改为迭代写法，判断规律不变，只是用循环更新当前节点，不再产生递归栈。

## 我的思维误区

- **默认 `q.val > p.val`**：参数顺序不代表数值顺序。先用 `Math.min` 和 `Math.max` 得到稳定的上下界，再进行判断。
- **尝试比较左右孩子与目标值**：真正决定搜索方向的是当前子树根 `root`。比较孩子会引入空指针风险，也不能直接表达两个目标是否仍在当前节点的同一侧。

## 一句话总结

> 把两个目标值整理成区间：当前根在区间左侧就向右，在区间右侧就向左，第一次落入区间的节点就是两条搜索路径的最低分叉位置。
