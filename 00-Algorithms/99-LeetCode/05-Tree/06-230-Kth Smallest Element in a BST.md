# 230. Kth Smallest Element in a BST

> 难度：Medium  
> 主题：Binary Search Tree、中序遍历、递归、顺序统计

## 1. 题目在问什么？

给定一棵二叉搜索树 BST 和整数 `k`，返回树中第 `k` 小的节点值。

`k` 使用 1-based index：

```text
k = 1 → 最小值
k = 2 → 第二小值
k = 3 → 第三小值
```

题目返回的是节点保存的 `int` 值，不是节点对象、节点层数或数组下标。

## 2. BST 提供了什么额外性质？

对于 BST 中的每个节点：

```text
左子树所有值 < 当前节点值
右子树所有值 > 当前节点值
```

例如：

```text
        5
      /   \
     3     8
    / \   / \
   2   4 7   9
```

所有节点值从小到大是：

```text
2, 3, 4, 5, 7, 8, 9
```

普通二叉树只提供左右孩子关系，无法保证左边的值一定更小；BST 的大小关系才是这道题能够使用有序遍历的关键。

## 3. 为什么中序遍历可以得到升序结果？

中序遍历的顺序是：

```text
左子树 → 当前节点 → 右子树
```

BST 的大小关系是：

```text
左子树值 < 当前节点值 < 右子树值
```

两者组合后：

```text
先访问更小的值
→ 再访问当前值
→ 最后访问更大的值
```

因此：

> BST 的中序遍历结果天然按照节点值升序排列。

这是本题最核心的转换：

```text
在 BST 中寻找第 k 小
→ 对 BST 做中序遍历
→ 取升序序列中的第 k 个元素
```

## 4. 中序遍历过程

考虑：

```text
        3
       / \
      1   4
       \
        2
```

从根节点 `3` 开始：

```text
先进入 3 的左子树
→ 到达节点 1
→ 1 没有左孩子
→ 访问 1
→ 进入 1 的右子树
→ 访问 2
→ 3 的左子树处理完成
→ 访问 3
→ 进入 3 的右子树
→ 访问 4
```

最终顺序：

```text
1, 2, 3, 4
```

如果 `k = 3`，目标就是这个升序序列中的第三个值。

## 5. 递归模板

中序遍历的递归结构是：

```java
public void dfs(TreeNode node) {
    if (node == null) {
        return;
    }

    dfs(node.left);
    // 在这里处理当前节点
    dfs(node.right);
}
```

三个常见 DFS 顺序的区别只在于“处理当前节点”的位置：

```text
前序：当前节点 → 左 → 右
中序：左 → 当前节点 → 右
后序：左 → 右 → 当前节点
```

本题必须使用中序的有序性质，所以将当前节点加入结果的位置放在两个递归调用之间：

```java
dfs(node.left);
res.add(node);
dfs(node.right);
```

## 6. 当前解法

```java
class Solution {
    List<TreeNode> res;

    public int kthSmallest(TreeNode root, int k) {
        res = new ArrayList<>();
        dfs(root);
        return res.get(k - 1).val;
    }

    public void dfs(TreeNode node) {
        if (node == null) {
            return;
        }

        dfs(node.left);
        res.add(node);
        dfs(node.right);
    }
}
```

整体过程：

```text
创建结果列表
→ 对 BST 进行完整中序遍历
→ 按升序保存所有 TreeNode
→ 取得第 k 个节点
→ 返回它的 val
```

## 7. 为什么是 `k - 1`？

题目中的 `k` 从 `1` 开始：

```text
第 1 小、第 2 小、第 3 小……
```

Java List 下标从 `0` 开始：

```text
下标 0、下标 1、下标 2……
```

对应关系是：

| 题目位置 | List 下标 |
| --- | --- |
| 第 1 小 | `0` |
| 第 2 小 | `1` |
| 第 3 小 | `2` |
| 第 k 小 | `k - 1` |

所以：

```java
res.get(k - 1)
```

获取第 `k` 小的节点，再通过：

```java
.val
```

返回节点值。

## 8. 为什么保存 `TreeNode`？

当前列表类型是：

```java
List<TreeNode> res;
```

所以中序遍历保存的是节点对象：

```java
res.add(node);
```

最终需要通过：

```java
res.get(k - 1).val
```

取得节点中的值。

如果只关心数值，也可以使用：

```java
List<Integer>
```

并在遍历时保存：

```java
res.add(node.val);
```

两种方式逻辑相同；保存节点对象可以保留完整节点信息，保存整数则更直接表达本题只需要节点值。

## 9. Base Case 为什么是 `node == null`？

```java
if (node == null) {
    return;
}
```

它表示当前方向已经走出了树。

例如一个叶子节点：

```text
  2
 / \
null null
```

执行：

```java
dfs(node.left);
```

会进入 `null`，立即返回；然后访问节点 `2`；接着右侧的 `null` 也立即返回。

Base Case 同时负责：

- 防止访问空节点；
- 终止递归；
- 让程序回到上一层继续执行。

## 10. 复杂度

当前解法会完整访问树中的每个节点一次。

设节点数量为 `n`：

```text
时间复杂度：O(n)
```

结果列表保存全部节点：

```text
结果空间：O(n)
```

递归调用栈的空间取决于树高 `h`：

```text
递归栈空间：O(h)
```

平衡 BST 中：

```text
h ≈ O(log n)
```

如果 BST 退化成链：

```text
h = O(n)
```

因此包含结果列表后，总额外空间是 `O(n)`。

## 11. 可以继续优化什么？

当前解法先遍历完整棵树，再取第 `k` 个节点。即使 `k = 1`，仍然会访问所有节点并保存全部结果。

由于中序遍历产生节点的顺序已经是从小到大，因此可以进一步思考：

```text
每访问一个节点就计数
→ 访问到第 k 个节点时立即记录答案
→ 不再保存完整中序列表
```

这可以将结果存储从 `O(n)` 降低到只保留递归栈所需的 `O(h)`。

如果能够在找到第 `k` 个节点后真正停止后续遍历，还可以避免访问不需要的较大节点。

不过当前完整中序遍历版本逻辑直观，是理解 BST 有序性质非常好的第一版解法。

## 12. 这道题能学到什么？

### 数据结构的性质决定遍历方式

如果输入只是普通二叉树，中序遍历不保证升序。

本题能这样处理，是因为：

```text
BST 的左小右大
+ 中序遍历的左根右
= 升序序列
```

看到 BST 时，应主动联想到中序遍历的有序性。

### Traversal Order 取决于处理时机

递归代码的结构非常相似，真正决定前序、中序、后序的是当前节点在哪个时刻被处理：

```text
进入节点立即处理       → 前序
左子树完成后处理       → 中序
左右子树都完成后处理   → 后序
```

### 1-based 与 0-based 需要转换

题目用自然语言描述“第 `k` 个”，通常从 `1` 开始；Java 数组和 List 下标从 `0` 开始。

因此经常需要：

```text
题目中的第 k 个 → Java 下标 k - 1
```

### 先写完整遍历，再考虑提前停止

做题时可以先完成清晰正确的版本：

```text
完整中序遍历
→ 得到升序列表
→ 取第 k 个
```

然后再观察：

```text
是否真的需要全部结果？
能否在得到目标后提前停止？
```

这是从正确解法走向空间和时间优化的常见过程。

## 13. 做题推导链条

```text
题目要求第 k 小
→ BST 满足左小右大
→ 中序遍历顺序是左、根、右
→ BST 的中序结果天然升序
→ 完整遍历并保存节点
→ List 下标从 0 开始
→ 第 k 小对应下标 k - 1
→ 返回该节点的 val
```

## 14. 一句话总结

> BST 的中序遍历会按照节点值升序访问所有节点，因此完整中序遍历后，第 `k` 小的节点就是列表下标 `k - 1` 对应的节点。
