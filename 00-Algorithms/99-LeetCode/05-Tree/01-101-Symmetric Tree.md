# 101. Symmetric Tree

## 题目

给一棵二叉树，判断它是不是左右对称。

左右对称的意思是：

```text
左子树和右子树像镜子一样对应。
```

不只是值要相等，结构也要对称。
## 这次 Wrong Answer 的原因

这次不是语法错误，也不是程序崩溃，而是逻辑判断错了。

错误的关键在于：你额外要求了某些子节点必须存在。

比如代码里类似这样的判断：

```java
if (right.right == null) {
    return false;
}
```

这句话的意思是：

```text
只要 right.right 是 null，就一定不是对称树。
```

但这是不对的。

叶子节点本来就没有孩子，所以它的 `left` 和 `right` 都是 `null`。如果两个镜像位置都是叶子节点，它们反而是对称的。

例如：

```text
        1
      /   \
     2     2
    / \   / \
   3   4 4   3
```

当递归比较到最下面两个 `3` 时：

```text
compare(3, 3)
```

这两个 `3` 都没有孩子：

```text
left.left == null
left.right == null
right.left == null
right.right == null
```

这是正常情况。

但如果代码里写了：

```java
if (right.right == null) {
    return false;
}
```

它就会把正常的叶子节点误判成错误。

## 树递归最重要的想法

写树的递归时，不要在当前层手动判断太多“孙子节点”。

当前函数只负责判断当前传进来的节点。

剩下的子树问题，交给下一层递归。

## 递归三要素

### 1. 函数定义

先想清楚函数到底表示什么。

对对称二叉树来说，可以定义一个辅助函数：

```text
compare(left, right)
```

它的含义是：

```text
判断 left 这棵树 和 right 这棵树 是否互为镜像。
```

注意：它不是只比较两个节点的值，而是比较两棵小树。

### 2. 终止条件

树题里最常见的终止条件是遇到 `null`。

比较两个节点时有三种情况：

```text
left == null && right == null
```

两个都是空，说明这一边是对称的。

```text
left == null && right != null
```

一个空，一个不空，不对称。

```text
left != null && right == null
```

一个不空，一个空，不对称。

所以不要害怕 `null`。

在树题里，`null` 是正常结构的一部分，不一定代表错误。

### 3. 递推关系

如果两个节点都不是 `null`，就看三件事：

```text
当前两个节点的值是否相等
左节点的左子树 是否 和 右节点的右子树 镜像
左节点的右子树 是否 和 右节点的左子树 镜像
```

也就是镜像位置要互相比较：

```text
外侧 vs 外侧
内侧 vs 内侧
```

图上看是这样：

```text
        left        right
        /  \        /   \
      LL   LR      RL   RR

需要比较：
LL 和 RR
LR 和 RL
```

## 这类题的提醒

不要这样想：

```text
我现在要把下面每一层都手动检查完。
```

要这样想：

```text
我当前层只处理当前节点。
下一层的问题，让递归函数自己处理。
```

树递归的关键不是写很多判断，而是把函数定义想清楚。

如果函数定义是：

```text
compare(left, right) 判断两棵树是否镜像
```

那你就不需要提前手动判断 `left.left.val`、`right.right.val` 这些细节。

因为下一次递归自然会比较它们。

## 一句话总结

对称树不是要求每个节点都有孩子，而是要求镜像位置的结构和值一致。
