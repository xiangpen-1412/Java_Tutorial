# 236. Lowest Common Ancestor of a Binary Tree

> 难度：Medium  
> 主题：Binary Tree、最近公共祖先、后序 DFS、递归返回值

## 问题

给定一棵普通二叉树以及树中两个不同的节点 `p` 和 `q`，寻找它们的最近公共祖先。祖先允许包含节点自身，因此如果 `p` 是 `q` 的祖先，答案就是 `p`，反之亦然。题目保证 `p` 和 `q` 都存在于树中。

与 [[10-235-Lowest Common Ancestor of a Binary Search Tree]] 不同，这里没有 BST 的大小关系，不能根据节点值判断应该进入哪一侧；最坏情况下必须检查整棵树。

## 核心收获

- **识别信号**：当前节点能否成为答案，取决于左、右子树分别发现了什么，因此要先获得子树结果，再判断当前节点——这是后序 DFS。
- **核心转换**：不必先保存 `root → p` 和 `root → q` 两条完整路径。每棵子树只向父层返回一个 `TreeNode`，用它概括当前子树的搜索结果。
- **通用方法**：递归返回值并不只表示最终答案；在不同阶段，它可以表示“找到的目标节点”，也可以表示“已经确定的最近公共祖先”。父层无需区分二者，只需继续向上传递。

## 递归契约

`lowestCommonAncestor(root, p, q)` 检查以 `root` 为根的子树，并返回：

```text
子树中 p、q 都没找到       → null
子树中只找到一个目标       → 返回该目标节点
子树中已经找到两个目标     → 返回它们的最近公共祖先
```

这个设计的精妙之处是把三种信息都压缩进 `TreeNode`：

```text
null      = 没有相关发现
非 null   = 发现了需要继续向上传递的关键节点
```

## 为什么是后序遍历

刚到达当前节点时，无法知道 `p` 和 `q` 位于哪里：

```text
都在左子树？
都在右子树？
分别位于两侧？
当前节点本身就是其中一个？
```

必须先取得左右子树的报告，再组合出当前层的结论：

```text
left  = 左子树的结果
right = 右子树的结果
最后判断当前 root 应该返回什么
```

因此这里的后序不只是访问顺序，而是一个依赖关系：

> 当前节点的结论依赖左右子树的结论。

## 四种组合情况

| `left` | `right` | 当前层的含义 |
| --- | --- | --- |
| `null` | `null` | 当前子树没有找到目标 |
| 非空 | `null` | 关键节点只来自左侧，继续向上传递 |
| `null` | 非空 | 关键节点只来自右侧，继续向上传递 |
| 非空 | 非空 | 两个目标在当前节点的两侧，当前节点是最低汇合点 |

“两侧找到目标”不是要求 `root.left == p` 且 `root.right == q`，而是两个目标分别存在于左右**子树中的任意深度**，并且 `p`、`q` 的左右顺序不固定。

## 当前节点等于目标时

递归遇到 `p` 或 `q` 时可以直接返回当前节点：

```text
另一个目标在当前节点下面
→ 当前节点本身就是最近公共祖先

另一个目标在当前节点所在子树之外
→ 当前节点先代表“已找到一个目标”，由祖先层继续组合
```

这两个场景使用同一个返回值即可表达，不需要在当前层预先知道另一个目标的位置。

## 结构骨架

```java
if (root == null || root == p || root == q) {
    return root;
}

TreeNode left = lowestCommonAncestor(root.left, p, q);
TreeNode right = lowestCommonAncestor(root.right, p, q);

if (left != null && right != null) {
    return root;
}

return left != null ? left : right;
```

最后一行同时覆盖三种情况：只有左边非空、只有右边非空、两边都为空。

## 为什么找到答案后不会丢失

如果某一层发现 `left` 和 `right` 都非空，它返回当前节点作为最近公共祖先。继续向上时，由于 `p` 和 `q` 已经全部包含在这个节点的子树中，其他分支不可能再找到新的 `p` 或 `q`。因此这个非空结果会被祖先层原样向上传递，直到成为整次调用的返回值。

## 从路径方案到后序方案

最初的方案是分别寻找两条根路径，再比较最后一个公共节点。这个方案是正确的，最坏时间也是 `O(n)`，但需要两次独立搜索并显式保存路径。

后序方案进一步发现：父节点真正需要的不是完整路径，而只是“这个子树有没有发现关键节点”。于是把路径信息压缩成递归返回值，一次遍历就能完成判断。

## 复杂度

- 时间复杂度：`O(n)`。普通二叉树没有有序性质，最坏情况下需要访问所有节点。
- 递归栈空间：`O(h)`，其中 `h` 是树高。
- 平衡树中 `h = O(log n)`；树退化成链表时 `h = O(n)`。

## 变种与注意点

- 语义上应使用 `root == p || root == q` 比较节点身份。使用 `root.val == p.val` 在本题节点值唯一的条件下也能判断，但不适用于允许重复值的二叉树。
- 这一简洁契约依赖题目保证两个目标都存在。如果允许目标缺失，仅返回一个 `TreeNode` 无法区分“只找到一个目标”和“已经得到合法答案”，需要返回更多状态。
- 若题目换成 BST，应优先利用大小关系排除整棵子树，不必遍历两侧。

## 我的思维演进

- **先寻找两条路径再比较**：逻辑正确，但保存了超过父层实际需要的信息。进一步追问“父层最少需要知道什么”，可以把两条路径压缩成子树返回值。
- **认为只有当前节点的左孩子是 `p`、右孩子是 `q` 时才成立**：目标不必是直接孩子，只需分别存在于左右子树中。判断依据是递归报告，而不是孩子节点本身。
- **认为从根向下无法立即确定答案**：这个观察是正确突破口。普通二叉树缺少方向信息，应改用后序思维，让结论从子树向上传递。

## 一句话总结

> 普通二叉树的 LCA 是后序汇总问题：子树返回自己发现的关键节点；左右都有发现时当前节点成为最低汇合点，否则把唯一的非空结果继续向上传递。
