# 04. 如何选择算法

## 1. 先不要从算法名开始

看到一道题，先不要问：

```text
这题是不是 DP？
这题是不是滑动窗口？
```

更好的顺序是：

```text
1. 题目要返回什么？
2. 输入是什么结构？
3. 暴力法会怎么做？
4. 暴力慢在哪里？
5. 我能维护什么、保存什么、换什么方向？
```

算法名是最后的结果，不是第一步。

## 2. Array

数组题通常在考：

```text
位置
顺序
区间
查找
原地修改
```

看到数组，先问：

```text
是否有序？
是否连续？
是否要原地修改？
是否要找两个/三个数？
是否要统计频率？
```

## 3. HashMap

当暴力法里有这种动作：

```text
每次都要找某个值以前有没有出现过
每次都要找 target - x
需要统计元素出现次数
需要把一种形式映射到一组结果
```

可以考虑 HashMap。

常见信号：

```text
Two Sum
frequency
anagram grouping
prefix sum count
seen before
```

核心思想：

```text
用空间换时间。
把重复查找变成 O(1) 查询。
```

## 4. Two Pointers

Two Pointers 通常用于：

```text
有序数组
左右夹逼
原地修改数组
反转
去重
```

### 左右指针

适合：

```text
数组有序
需要找两个数
可以根据 sum 太大/太小移动指针
```

比如：

```text
Two Sum II
3Sum
```

判断方式：

```text
如果数组有序，且我能根据当前结果决定 left++ 还是 right--，就想左右指针。
```

### 快慢指针 / 读写指针

适合：

```text
原地删除
原地去重
移动元素
压缩数组
```

比如：

```text
Remove Duplicates
Move Zeroes
```

核心思想：

```text
right 负责扫描所有元素
left 负责维护有效区域
```

## 5. Binary Search

Binary Search 用在：

```text
有一个候选范围
每次可以判断答案在左边还是右边
每次可以排除一半
```

常见信号：

```text
sorted / ascending
查找某个值
插入位置
第一个满足条件的位置
最后一个满足条件的位置
最小可行答案
最大可行答案
答案范围很大
```

核心不是“有序数组”，而是：

```text
能不能通过 mid 判断一半一定没答案。
```

普通有序数组只是最直观的二分。

旋转数组、二维矩阵、答案空间也可能是二分，因为它们仍然保留了某种可以排除一半的性质。

注意：

```text
判断性质时，可以带上 mid。
移动搜索范围时，通常排除 mid。
```

比如旋转数组：

```text
判断哪边有序：看 left 到 mid 或 mid 到 right
缩小搜索范围：left = mid + 1 或 right = mid - 1
```

## 6. Sliding Window

Sliding Window 用在：

```text
连续子数组
连续子字符串
窗口可以扩张和收缩
窗口状态可以维护
```

常见关键词：

```text
substring
subarray
continuous
longest
shortest
minimum window
at most / at least
```

适合的情况：

```text
窗口右边扩大时，可以更新状态
窗口不合法时，可以移动左边让它重新合法
```

常见维护内容：

```text
字符频率
窗口长度
窗口和
重复元素
满足条件的数量
```

注意：

```text
不是所有 subarray sum 都能用 sliding window。
```

如果数组里有负数，sum 可能忽大忽小，窗口扩大/缩小不再稳定。

这种时候通常更适合：

```text
Prefix Sum
```

## 7. Prefix Sum

Prefix Sum 用在：

```text
连续区间的累计和
多次查询区间和
统计有多少个 subarray 的和满足条件
```

常见关键词：

```text
range sum
subarray sum
sum equals k
continuous
```

核心公式：

```text
区间和 = 两个 prefix sum 的差
```

什么时候只用 prefix array：

```text
题目直接给 left 和 right，让你求区间和。
```

什么时候 prefix sum + HashMap：

```text
题目不告诉 left/right，而是让你统计有多少个区间和等于 k。
```

这时 HashMap 存：

```text
prefix sum -> 出现次数
```

## 8. Prefix / Suffix

Prefix / Suffix 思想用于：

```text
每个位置的答案需要左边信息和右边信息
```

比如：

```text
Product of Array Except Self
Trapping Rain Water
```

判断方式：

```text
当前位置需要左边累计结果，也需要右边累计结果。
```

思路：

```text
左边信息从左往右维护
右边信息从右往左维护
```

## 9. Stack

Stack 用在：

```text
最近的东西要最先处理
需要匹配
需要回退
需要保存还没解决的元素
```

常见题型：

```text
括号匹配
表达式计算
Min Stack
单调栈
```

### 普通 Stack

适合：

```text
括号匹配
撤销操作
函数调用模拟
```

核心：

```text
Last In First Out
```

### Monotonic Stack

适合：

```text
下一个更大元素
下一个更小元素
右边第一个更大
等待未来某个更大的值
```

常见关键词：

```text
next greater
daily temperatures
first greater on the right
```

核心思想：

```text
栈里存还没找到答案的元素。
当遇到一个能解决它们的新元素时，弹出并计算答案。
```

## 10. Linked List

Linked List 题重点不是值，而是：

```text
节点之间的连接关系
```

常见信号：

```text
reverse
cycle
middle
nth from end
intersection
merge
```

常用工具：

```text
dummy node
prev / curr / next
fast / slow
HashSet
```

什么时候用快慢指针：

```text
找中点
判断环
倒数第 n 个节点
```

什么时候用 dummy：

```text
可能删除头节点
需要统一处理链表头
构造新链表
```

## 11. Tree

Tree 题通常用：

```text
DFS
BFS
Recursion
```

看到输入是：

```text
TreeNode root
```

先想：

```text
我在当前节点要做什么？
我要从左子树/右子树拿什么信息？
我要把什么信息传给子树？
```

### DFS

适合：

```text
深度
高度
路径
左右子树关系
BST 判断
递归比较结构
```

常见问题：

```text
Maximum Depth
Path Sum
Symmetric Tree
Validate BST
Balanced Binary Tree
```

### BFS

适合：

```text
层序遍历
按层输出
最短层数
从上到下一层层扩散
```

常用结构：

```text
Queue
```

## 12. Backtracking

Backtracking 用在：

```text
返回所有方案
每一步都有多个选择
需要试完一条路再退回来
```

常见关键词：

```text
all
all possible
generate
combinations
permutations
subsets
valid boards
```

核心动作：

```text
做选择
递归
撤销选择
```

写之前先问：

```text
path 是什么？
choice 是什么？
什么时候记录答案？
怎么避免重复？
怎么撤销？
```

常见区别：

```text
Subsets: 顺序不重要，用 start，递归传 i + 1
Permutations: 顺序重要，每层从头选，用 used
Combination Sum: 顺序不重要，但数字可重复，用 start，递归传 i
```

## 13. DP

DP 用在：

```text
最大值
最小值
多少种方法
能不能达到
当前结果依赖之前结果
暴力递归会重复计算子问题
```

常见关键词：

```text
maximum
minimum
ways
can reach
choose or not choose
adjacent restriction
```

DP 四问：

```text
1. dp[i] 表示什么？
2. 初始状态是什么？
3. 状态转移是什么？
4. 最后返回什么？
```

### Top-down

适合：

```text
递归关系更容易想
从目标问题往小问题拆
```

形式：

```text
递归 + memo
```

### Bottom-up

适合：

```text
小状态到大状态的顺序很清楚
可以从前往后填表
```

形式：

```text
循环 + dp array
```

### 状态压缩

如果当前状态只依赖前几个状态：

```text
dp[i - 1]
dp[i - 2]
```

可以考虑：

```text
不用完整 dp array，只用几个变量。
```

## 14. Greedy / 状态维护

有些题不一定要硬叫 DP。

如果每一步只需要维护：

```text
目前最小值
目前最大值
目前最优答案
```

可能是：

```text
Greedy / 状态维护
```

比如股票题：

```text
固定卖出日
维护过去最低买入价
维护最大利润
```

核心不是标签，而是：

```text
当前答案需要什么历史信息？
这个历史信息能不能用变量维护？
```

## 15. 换遍历方向

当你发现：

```text
我固定当前位置，但需要知道右边未来的信息
```

可以考虑：

```text
从右往左扫
```

当你发现：

```text
我需要左边过去的信息
```

可以考虑：

```text
从左往右扫
```

如果左右都要：

```text
两遍扫描
prefix + suffix
```

记忆：

```text
未来信息不好拿。
换方向后，它就变成已经见过的信息。
```

## 16. 最后判断流程

做题时可以按这个顺序：

```text
1. 输入结构是什么？
   array / string / linked list / tree / graph

2. 题目要什么？
   一个值 / 所有方案 / 数量 / 是否存在 / 最大最小

3. 暴力怎么做？
   枚举什么？

4. 暴力慢在哪里？
   重复查找 / 重复求和 / 重复子问题 / 重复扫描

5. 能维护什么？
   seen map / prefix sum / dp / window / stack / min / max

6. 信息在哪边？
   左边 / 右边 / 左右都要

7. 最后再确定算法名。
```

## 17. 一句话总结

```text
HashMap: 重复查找
Two Pointers: 有序夹逼或原地修改
Binary Search: 每次排除一半
Sliding Window: 连续区间可扩可缩
Prefix Sum: 连续区间累计和
Stack: 最近的未解决元素
LinkedList: 指针连接关系
Tree: 子树递归或层序遍历
Backtracking: 所有方案
DP: 当前答案由之前答案推出
Greedy/状态维护: 维护历史最优信息
```
