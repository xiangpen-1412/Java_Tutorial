## 阶段总结

### 已经比较稳的能力

- 基础数据结构题型已经完整走过一轮：数组、链表、栈、队列、树。
- HashMap 类题目已经从 Two Sum 进到 Group Anagrams、Top K、Prefix Sum 计数。
- 双指针不只是会写左右指针，也已经练过快慢指针和原地覆盖。
- 滑动窗口已经接触到 Minimum Window Substring 这种高频难题。
- 单调栈已经从 Daily Temperatures 进到 Next Greater Element II。
- 树已经从基础递归、BFS 进到 BST 验证和平衡树判断。
- 回溯和 DP 已经完成入门模板，下一步可以开始做更典型的中等题。

### 下一阶段重点

下一批不要只追求题量，而是开始练“组合题型”和“模板识别”：

- 二分不只是在数组里找数，还要练旋转数组和答案空间。
- 堆要和 HashMap / Top K 结合起来。
- 图论要正式开始，重点是 BFS / DFS + visited。
- 树要从单纯遍历进入“递归返回值设计”。
- DP 要从一维入门进入选择型、路径型、序列型。

### 做题顺序原则

以后每一批题都按这个顺序排：

1. 先做前置小模型题。
2. 再做同模板的标准题。
3. 最后做综合题或高频难题。

比如 `84. Largest Rectangle in Histogram` 不能直接放在 `503. Next Greater Element II` 后面硬做。`503` 练的是 next greater element，但 `84` 真正需要的是：

- next smaller element
- previous smaller element
- 单调递增栈
- 用左右边界计算宽度

所以这类题必须先补“小问题”，再做“大问题”。
