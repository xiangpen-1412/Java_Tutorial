# 02. 固定哪个点与遍历方向

## 1. 思路转变的核心

很多数组题不是难在代码，而是难在：

```text
我到底固定哪个位置？
我需要的信息在左边还是右边？
```

如果固定的位置选得不合适，就会变成每次都要重新扫描。

如果换一个固定点，或者换一个遍历方向，信息就可以被一个变量维护。

## 2. 问题模板

以后遇到一维数组题，可以问：

```text
1. 我现在固定的是哪个点？
2. 固定这个点时，我还需要知道什么？
3. 这个信息在它左边还是右边？
4. 当前遍历方向能不能顺手维护这个信息？
5. 如果不能，能不能换方向？
```

## 3. 信息在左边

如果当前点需要的是左边信息：

```text
左边最小值
左边最大值
左边累计和
左边出现过的数字
```

通常可以从左往右扫。

因为走到当前点时，左边已经看过了。

可以维护：

```text
leftMin
leftMax
prefixSum
HashMap
```

## 4. 信息在右边

如果当前点需要的是右边信息：

```text
右边最大值
右边最小值
右边累计乘积
右边下一个更大元素
```

从左往右时，右边是未来，不好知道。

这时候可以考虑：

```text
从右往左扫
```

走到当前点时，右边已经看过了。

可以维护：

```text
rightMax
rightMin
suffixProduct
stack
```

## 5. 股票题里的转变

股票题本质是找：

```text
buy day < sell day
让 sell price - buy price 最大
```

### 视角 1：固定卖出日

如果固定今天卖：

```text
今天卖能赚多少？
```

需要知道：

```text
今天之前最低的买入价
```

这是左边信息。

所以可以：

```text
从左往右扫
维护 leftMin
维护 maxProfit
```

### 视角 2：固定买入日

如果固定今天买：

```text
今天买，未来最好哪天卖？
```

需要知道：

```text
今天之后最高的卖出价
```

这是右边信息。

所以可以：

```text
从右往左扫
维护 rightMax
维护 maxProfit
```

两个视角都能优化。

关键是：

```text
不要每次固定一天后，再重新扫描另一边。
```

## 6. Product of Array Except Self 里的转变

每个位置需要：

```text
左边所有数的乘积
右边所有数的乘积
```

左边信息：

```text
从左往右维护
```

右边信息：

```text
从右往左维护
```

所以这类题自然会想到：

```text
prefix + suffix
```

## 7. 什么时候要换方向？

当你发现自己在想：

```text
对每个 i，我都要去它右边找一个东西
```

就应该停下来问：

```text
能不能从右往左？
```

当你发现自己在想：

```text
对每个 i，我都要去它左边找一个东西
```

就问：

```text
能不能从左往右维护？
```

如果左右都要：

```text
可能需要两遍，或者 prefix/suffix。
```

## 8. 记忆

```text
未来信息不好拿。
换个方向，它就变成已经见过的信息。
```

```text
固定买入日，需要未来最高卖价。
固定卖出日，需要过去最低买价。
```

```text
方向不是随便选的。
方向取决于你要维护的信息在哪一边。
```
