# 05. 贡献法

## 核心理解

贡献法的本质是：

```text
不要枚举每个结果对象。
改成枚举每个元素对答案的贡献。
```

当题目表面要求统计很多对象时，比如：

```text
所有 subarray
所有 rectangle
所有 interval
所有 pair
```

直接枚举这些对象可能太贵。

这时可以换一个角度：

```text
每个元素能负责多少个对象？
每个元素会被计算进答案多少次？
```

如果能算出每个元素的贡献，就可以把答案变成：

```text
总答案 = 每个元素的贡献之和
```

## 什么时候想到贡献法

看到这些信号，可以考虑贡献法：

```text
题目要统计所有区间 / 子数组 / 矩形 / pair
直接枚举对象会超时
答案是 sum / count / max / min
每个结果对象的关键值由某个元素决定
题目里有 minimum / maximum / largest / smallest / bottleneck
```

尤其是当一个对象的结果由某个元素“负责”时：

```text
一个 subarray 的 minimum 一定是其中某个元素。
一个 rectangle 的高度一定由某个最矮柱子决定。
一个区间的 maximum 一定是其中某个元素。
```

这时就可以反过来问：

```text
这个元素作为关键元素时，可以负责多少个对象？
```

## 思考模板

```text
1. 题目让我统计很多对象吗？
   subarray / rectangle / interval / pair

2. 直接枚举这些对象会不会太多？

3. 每个对象的答案是不是由某个元素决定？
   min / max / shortest / tallest / bottleneck

4. 能不能反过来问：
   每个元素能负责多少个对象？

5. 要知道这个元素负责的范围，需要什么边界？
   左边第一个更小 / 更大
   右边第一个更小 / 更大

6. 如果需要快速找左右第一个更小 / 更大，就考虑 Monotonic Stack。
```

## 例子：907. Sum of Subarray Minimums

题目表面问：

```text
所有 subarray 的 minimum 加起来是多少？
```

直接想法是：

```text
枚举每个 subarray
计算它的 min
加起来
```

这个视角是：

```text
以 subarray 为中心
```

但是 subarray 有 `O(n^2)` 个，不能直接枚举。

换成贡献法：

```text
以每个元素 arr[i] 为中心
```

问：

```text
arr[i] 会在哪些 subarray 里成为最小值？
它一共作为最小值出现多少次？
```

如果 `arr[i]` 作为最小值出现了 `count` 次，那么它的贡献是：

```text
arr[i] * count
```

总答案就是：

```text
sum(arr[i] * count[i])
```

这里的关键问题会变成：

```text
arr[i] 作为最小值时，左边最多扩到哪里？
右边最多扩到哪里？
```

这就需要找边界：

```text
左边第一个更小元素
右边第一个更小元素
```

所以这题最后会和 Monotonic Stack 连起来。

## 例子：84. Largest Rectangle in Histogram

题目表面问：

```text
最大的 rectangle 面积是多少？
```

直接枚举 rectangle 很难。

换成贡献法：

```text
枚举每个柱子 heights[i] 作为矩形的最低高度。
```

因为任何一个矩形的高度一定由其中最矮的柱子决定。

所以问：

```text
如果 heights[i] 是最低高度，
它最多能往左扩多远？
最多能往右扩多远？
```

面积就是：

```text
heights[i] * width
```

这里也会变成边界问题：

```text
左边第一个更矮柱子
右边第一个更矮柱子
```

所以也适合 Monotonic Stack。

## 为什么会连接到单调栈

贡献法经常会问：

```text
当前元素作为 min / max / bottleneck 时，可以控制多大范围？
```

范围通常由“第一个破坏条件的元素”决定。

比如：

```text
作为 minimum：
往左 / 往右直到遇到更小的元素。

作为 maximum：
往左 / 往右直到遇到更大的元素。

作为 histogram 最低高度：
往左 / 往右直到遇到更矮的柱子。
```

这类问题本质是：

```text
Previous Smaller / Greater
Next Smaller / Greater
```

所以自然想到：

```text
Monotonic Stack
```

## 记忆

```text
枚举对象太贵，就枚举贡献者。
```

```text
不要问：每个 subarray 的 min 是什么？
要问：每个元素能作为多少个 subarray 的 min？
```

```text
不要问：每个 rectangle 面积是多少？
要问：每个柱子作为最低高度时，能形成多大 rectangle？
```

一句话：

```text
贡献法 = 换统计对象。
```
