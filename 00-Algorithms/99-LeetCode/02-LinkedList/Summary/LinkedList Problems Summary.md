# LinkedList 已做题目总结：移动引用、维护连接关系

链表题的核心不是下标，而是节点对象和 `next` 引用。做题时要持续问：当前指针指向谁、修改 `next` 后是否还找得到后续节点、最终应该返回哪个节点。

## 2. 链表操作的第一原则：先保存，再断开

反转链表时，如果直接执行：

```java
current.next = previous;
```

原来的后续链表可能丢失。因此标准思考顺序是：

```text
保存 next
修改 current.next
previous 前进
current 前进
```

这不仅适用于反转，也适用于删除、插入和拆分链表。

## 3. Dummy Node：统一头节点边界

dummy 是放在真实头节点之前的虚拟节点：

```text
dummy → head → ...
```

当前笔记中主要用于：

```text
2：逐步构造新的结果链表
19：删除节点时，让删除头节点和删除中间节点使用同一逻辑
```

它解决的是“结果头节点尚未确定”或“头节点可能被删除”的特殊情况。

常见搭配：

```text
dummy 固定保存结果入口
current / tail 负责向后追加节点
最终返回 dummy.next
```

## 4. 快慢指针的三种用途

### 找中点

```text
slow 每次走一步
fast 每次走两步
fast 到末尾时，slow 位于中间
```

对应 [[05-876-Middle of the Linked List]]，也是回文链表拆分前后半段的基础。

### 判断环

有环时，快指针进入环后会追上慢指针；无环时，快指针先到 `null`。

比较的是节点对象：

```java
slow == fast
```

不能只比较 `val`，因为不同节点可以保存相同值。

### 保持固定距离

删除倒数第 `n` 个节点时，让 fast 先走出固定间隔，再让 fast、slow 同时移动。fast 到末尾时，slow 停在待删除节点之前。

## 5. 指针换头：自动补齐长度差

[[07-160-Intersection of Two Linked Lists]] 中，两条链表长度可能不同：

```text
指针 A 走完 A 后转到 B
指针 B 走完 B 后转到 A
```

两者都走过：

```text
lengthA + lengthB
```

长度差被自动抵消。若相交，会在同一个节点对象相遇；不相交则最终同时为 `null`。

## 6. 局部模拟：Add Two Numbers

不需要先把整条链表转换成一个大整数。每一轮只处理当前两位和进位：

```text
sum = x + y + carry
当前位 = sum % 10
新进位 = sum / 10
```

循环条件需要覆盖三种剩余信息：

```text
l1 还有节点
l2 还有节点
carry 仍不为 0
```

## 7. 组合操作：Palindrome Linked List

回文链表不是一个全新的基础结构，而是已有操作的组合：

```text
快慢指针找中点
反转后半段
前半段与反转后的后半段同步比较
```

它训练的是把多个链表基本动作连接起来，而不是再背一种独立模板。

## 8. 选择方法

| 题目信号 | 优先考虑 |
| --- | --- |
| 修改整条链的方向 | `previous/current/next` |
| 构造新的链表结果 | dummy + tail |
| 头节点可能被删除 | dummy |
| 中点、环、倒数位置 | 快慢或间隔双指针 |
| 两条链长度不同但要对齐 | 指针换头 |
| 两条链逐位计算 | 同步遍历 + carry |
| 比较链表前后两半 | 中点 + 反转 + 同步比较 |

## 9. 常见错误

```text
修改 next 前没有保存后续节点；
移动了 head，最后丢失原入口；
循环条件检查的变量没有在循环中变化；
把“节点相同”写成“val 相同”；
删除节点时停在了待删除节点，而不是它的前一个节点；
fast.next.next 前没有先检查 fast 和 fast.next；
构造结果时返回 current，而不是 dummy.next。
```

## 10. 记忆主线

```text
链表没有随机访问，所有信息都来自指针移动。
先保存后续，再修改连接。
边界复杂时使用 dummy。
位置关系问题优先考虑双指针速度或间隔。
比较相交、成环时关注节点对象，不是节点值。
```
