# LeetCode 206：反转链表背后的知识  
  
## 题目在考什么  
  
给你一个单链表，让你把链表的方向反过来。  
  
原来是：  
  
```text  
1 -> 2 -> 3 -> 4 -> 5  
```  
  
反转后变成：  
  
```text  
5 -> 4 -> 3 -> 2 -> 1  
```  
  
这道题不只是考代码，而是考你是否真正理解链表的节点关系。  
  
---  
  
## 背后的核心知识  
  
链表和数组不一样。  
  
数组是靠下标访问：  
  
```text  
nums[0], nums[1], nums[2]  
```  
  
链表是靠节点之间的引用连接：  
  
```text  
node.next  
```  
  
所以链表反转的本质不是“交换位置”，而是：  
  
> 改变每个节点的 `next` 指向。  
  
节点本身没有移动，也没有消失，只是它们之间的连接方向变了。  
  
---  
  
## 这道题真正要学什么  
  
### 1. 理解 `next` 是指向关系  
  
`head.next` 不是下一个节点的副本，而是当前节点指向下一个节点的引用。  
  
修改 `head.next`，就是在修改链表结构。  
  
### 2. 修改链表前要保存后续节点  
  
如果你直接改变当前节点的 `next`，可能会丢失后面的链表。  
  
所以链表题里经常需要先保存：  
  
```java  
ListNode next = curr.next;  
```  
  
这一步的意义是：  
  
> 先记住后面是谁，再改变当前节点的指向。  
  
### 3. 链表题重点是指针关系  
  
很多链表题不是难在语法，而是难在你脑子里要清楚：  
  
```text  
当前节点是谁？  
前一个节点是谁？  
后一个节点有没有被保存？  
修改 next 之后链表会不会断？  
```  
  
---  
  
## 一句话总结  
  
LeetCode 206 的核心是：  
  
> 链表反转不是移动节点，而是把每个节点的 `next` 指向反过来。
