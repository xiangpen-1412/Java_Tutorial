# LeetCode 19：Remove Nth Node From End 背后要学什么

## 1. 这道题在考什么

这道题给你一个单链表 `head`，让你删除倒数第 `n` 个节点。

表面上看，它是在问：

```text
怎么找到倒数第 n 个节点？
```

但真正要考的是：

```text
你能不能在链表里找到“要删除节点的前一个节点”，然后改掉它的 next？
```

链表删除题的核心通常不是删除当前节点，而是找到谁指向了当前节点。

因为单链表只能往后走，当前节点不知道自己的前一个节点是谁。

所以这道题真正的关键点是：

> 删除一个节点，本质上是修改它前一个节点的 `next` 指向。

---

## 2. 我这次一开始的思路是对的

我一开始想到的是：

```text
先遍历一遍链表，得到总长度 length。
然后算出倒数第 n 个节点，对应正数第几个节点。
再找到它前面的那个节点，把连接关系改掉。
```

这个大方向完全没问题。

我真正卡住的地方不是算法方向，而是指针关系被自己绕乱了。

尤其是我一开始想用：

```text
copy
dummy
dummy2
```

去重新拼一个链表。

这导致我脑子里同时出现了两套逻辑：

```text
一套是：在原链表上删除节点
另一套是：重新构造一个结果链表
```

这两套思路混在一起后，就很容易出现：

```text
节点没有真正跳过
dummy 和 copy 指向同一个节点
某个节点的 next 指向自己
链表产生环
遍历时 infinite loop
```

所以这次最重要的收获是：

> 链表题不要急着写多个指针，先想清楚每个指针到底代表什么角色。

---

## 3. 我为什么会被 dummy 绕进去

我之前写的 `dummy` 其实有时候不是“虚拟头节点”，而只是一个普通遍历指针。

这两个概念要分清：

```text
真正的 dummy node：
新建出来的一个假头节点，用来统一处理删除头节点等边界情况。

普通遍历指针：
只是指向链表里的某个节点，用来往后走。
```

如果只是写：

```text
dummy = head
```

那它不是真 dummy。

它只是另一个变量，也指向原来的头节点。

真正的 dummy node 应该是一个新节点，它的作用不是保存数据，而是站在原链表头节点前面，帮我处理“删除第一个节点”的特殊情况。

以后看到自己写了 `dummy`，要问一句：

```text
这个 dummy 到底是一个新建的假头节点？
还是只是 head 的另一个名字？
```

如果这个问题没想清楚，后面的指针就很容易乱。

---

## 4. 这道题让我理解了“删除节点”到底是什么意思

链表里的节点并不是靠位置存在的。

它们靠的是引用连接：

```text
1 -> 2 -> 3 -> 4 -> 5
```

如果要删除 `4`，并不是把 `4` 这个对象从内存里手动抹掉。

真正发生的是：

```text
让 3 不再指向 4，而是直接指向 5。
```

这样从 `head` 出发时，就走不到 `4` 了。

所以链表删除题最核心的问题是：

```text
我要删谁？
谁的 next 正在指向它？
这个 next 应该改成谁？
```

只要这三个问题能回答清楚，代码通常不会长。

---

## 5. 为什么会出现 infinite loop

我之前的问题之一是，在重新拼链表时，把 `dummy` 和 `copy` 移到了同一个节点。

如果下一轮又执行类似：

```text
dummy.next = copy
```

而此时 `dummy` 和 `copy` 是同一个节点，就等于：

```text
这个节点的 next 指向它自己
```

链表就变成了：

```text
5 -> 5 -> 5 -> 5 -> ...
```

这就是 infinite loop 的来源。

这说明链表题里最危险的事情之一是：

> 在没有想清楚指针身份时，随便改 `next`。

数组里下标错了，可能只是取错值。

链表里 `next` 改错了，可能直接把结构改坏，产生断链或者环。

---

## 6. 这道题背后想训练我什么

这道题不是单纯想让我背一个写法。

它想训练的是几个链表基本能力：

```text
1. 能把“倒数第 n 个”转换成一个可以定位的位置。
2. 明白删除节点要操作的是前一个节点。
3. 能处理删除头节点这种边界情况。
4. 能区分“遍历指针”和“虚拟头节点”。
5. 能意识到链表不是数组，不能靠 index 直接跳过去。
```

更深一点，它还在问：

```text
你能不能把链表看成一组 next 连接关系？
你能不能只改必要的一条连接，而不是想着重建整个链表？
```

这也是我这次最大的思维转变：

> 原链表不用重建，很多时候只需要改一根箭头。

---

## 7. 以后遇到类似链表删除题应该怎么想

以后看到链表删除题，不要先急着写代码。

先问自己这几个问题：

```text
我要删除的是哪个节点？
我能不能找到它的前一个节点？
如果要删除的是 head 怎么办？
我改完 next 之后，后面的链表还连着吗？
我有没有可能让某个节点指向自己？
```

如果题目里出现：

```text
倒数第 n 个
中间节点
是否有环
链表后半段
```

就要想到链表常用的双指针思想。

数组里的 two pointers 经常靠下标移动。

链表里的 fast / slow pointers 经常靠 `next` 移动。

它们本质都是：

> 用两个位置之间的距离关系，帮助我定位某个特殊节点。

---

## 8. 我这道题真正学到的东西

这道题我不是完全没思路。

我已经想到了：

```text
先算长度
再找到要删除的位置
再修改前一个节点的 next
```

这说明大方向是对的。

真正需要练的是：

```text
把口头思路翻译成更简单的指针操作。
不要把原地修改和重新构造混在一起。
不要因为变量名叫 dummy，就默认它真的是 dummy node。
删除节点时，重点看“前一个节点”。
改 next 之前，脑子里先画清楚当前节点关系。
```

以后我写链表题时，可以先在纸上画三个东西：

```text
prev：要删除节点的前一个
target：要删除的节点
next：删除后要接上的节点
```

只要这三个角色清楚了，链表删除题就不会那么吓人。

---

## 一句话总结

LeetCode 19 真正要我学的不是某一段代码，而是：

> 链表删除的本质是找到目标节点的前一个节点，然后只修改那一条 `next` 连接；不要把“原地删除”和“重新拼链表”混在一起。
