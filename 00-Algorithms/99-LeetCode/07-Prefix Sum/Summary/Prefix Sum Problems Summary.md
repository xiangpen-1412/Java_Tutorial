# Prefix Sum 已做题目总结：把区间信息转换成两个前缀状态

前缀思想的核心是预先累计“从开头到当前位置”的信息，使一个区间可以通过两个前缀状态组合得到。当前目录还包含前缀积/后缀积，它与前缀和共享“先保存局部累计结果”的思想。

## 1. 已覆盖的题型

| 题目 | 题目类型 | 主要方法 | 核心公式/状态 |
| --- | --- | --- | --- |
| [[01-303-Range Sum Query - Immutable]] | 多次查询固定数组区间和 | 前缀和 | `sum(left,right) = prefix[right+1] - prefix[left]` |
| [[02-560-Subarray Sum Equals K]] | 统计和为 k 的连续子数组 | 前缀和 + HashMap 计数 | 查找 `currentPrefix - k` |
| [[03-238-Product of Array Except Self]] | 除自己外的乘积 | 前缀积 + 后缀积 | 左侧乘积 × 右侧乘积 |

## 2. 前缀数组为什么多开一个位置？

定义：

```text
prefix[i] = 前 i 个元素之和
prefix[0] = 0
```

那么数组区间 `[left, right]` 的和是：

```text
prefix[right + 1] - prefix[left]
```

多出的 `prefix[0]` 让从下标 0 开始的区间不需要特殊判断。

## 3. 303：预处理换取重复查询效率

如果数组不变但要多次查询区间和：

```text
构建 prefix：O(n)
每次查询：O(1)
```

这是典型的空间换时间：把重复计算提前完成。

识别信号：

```text
同一个数组
大量区间查询
数组查询期间不修改
```

## 4. 560：把区间条件变成历史前缀查询

设当前前缀和为 `currentPrefix`，某个更早前缀为 `previousPrefix`：

```text
区间和 = currentPrefix - previousPrefix
```

希望区间和等于 `k`：

```text
currentPrefix - previousPrefix = k
previousPrefix = currentPrefix - k
```

所以遍历到当前位置时，只需知道过去出现过多少次 `currentPrefix - k`。

HashMap 保存：

```text
key   = 某个前缀和
value = 该前缀和出现次数
```

初始化必须包含：

```text
前缀和 0 出现 1 次
```

它代表空前缀，使从下标 0 开始且和为 `k` 的区间能被统计。

## 5. 为什么 Map 保存次数而不是是否存在？

题目要求子数组数量。同一个前缀和值可能在多个位置出现，每个位置都能形成不同子数组：

```text
Set：只能回答出现过没有
Map<sum, count>：能回答有多少个合法起点
```

更新顺序通常是先查询历史，再把当前前缀加入 Map，避免错误使用当前位置本身。

## 6. 238：前缀与后缀的组合

对位置 `i`：

```text
answer[i] = i 左边所有数的乘积 × i 右边所有数的乘积
```

处理方式：

```text
从左到右保存每个位置左侧乘积
从右到左维护不断累积的右侧乘积
两者相乘得到答案
```

它不使用除法，因此自然处理数组中存在 0 的情况。

## 7. 选择方法

| 题目信号 | 优先考虑 |
| --- | --- |
| 固定数组的大量区间和查询 | 前缀和数组 |
| 统计满足某个和的连续子数组数量 | 前缀和 + HashMap 次数 |
| 每个位置需要左侧累计信息与右侧累计信息 | 前缀状态 + 后缀状态 |

## 8. 常见错误

```text
prefix 下标定义不清，right 是否加一混乱；
忘记 prefix[0] = 0；
560 使用滑动窗口处理含负数数组，窗口不再具有单调性；
Map 只保存是否存在，丢失重复前缀的贡献；
先更新 Map 再查询，统计了不应出现的空区间；
238 直接使用总乘积除以当前值，遇到 0 出错。
```

## 9. 记忆主线

```text
区间和 = 右前缀 - 左前缀
区间和等于 k → 查历史前缀 current - k
统计数量 → Map 保存出现次数
除自己以外 → 左侧累计 × 右侧累计
```
