# 238. Product of Array Except Self

## 题目

给一个整数数组 `nums`，返回一个数组 `answer`。

其中：

```text
answer[i] = nums 里除了 nums[i] 自己以外，其他所有数字的乘积。
```

题目要求不能使用除法，并且希望在线性时间内完成。
## 1. 这题学到的核心

这题学到的是：

```text
一个位置的答案，可以拆成左边累计结果和右边累计结果。
```

题目要的是：

```text
answer[i] = 除了 nums[i] 自己以外，其他所有元素的乘积
```

所以对每个位置 `i` 来说：

```text
answer[i] = i 左边所有数的乘积 * i 右边所有数的乘积
```

这就是：

```text
prefix product + suffix product
```

## 2. 为什么不能只想 total product？

第一反应可能是：

```text
先算所有数字的乘积 total
answer[i] = total / nums[i]
```

但题目限制：

```text
不能使用除法
```

而且数组里可能有 `0`，除法思路也会变复杂。

所以这题真正要练的是：

```text
不用除法，如何用左右两边的累计乘积得到答案。
```

## 3. 暴力为什么慢？

暴力思路是：

```text
对每个 i，都重新把除了 nums[i] 以外的所有数乘一遍。
```

这样会重复计算很多次。

比如：

```text
nums = [1, 2, 3, 4]
```

算 `answer[2]` 时会乘：

```text
1 * 2 * 4
```

算 `answer[3]` 时又会乘：

```text
1 * 2 * 3
```

左边的累计乘积被反复算。

优化思路就是：

```text
把左边累计乘积和右边累计乘积提前维护起来。
```

## 4. Prefix Product 是什么？

Prefix product 表示：

```text
当前位置左边所有数字的乘积
```

注意这题里的左边乘积不包括自己。

比如：

```text
nums = [1, 2, 3, 4]
```

每个位置左边乘积是：

```text
index 0: 左边没有数字 -> 1
index 1: 1
index 2: 1 * 2
index 3: 1 * 2 * 3
```

这里左边没有数字时用 `1`，因为：

```text
1 是乘法里的 identity。
```

就像加法里空和可以看成 `0`，乘法里空乘积可以看成 `1`。

## 5. Suffix Product 是什么？

Suffix product 表示：

```text
当前位置右边所有数字的乘积
```

同样不包括自己。

比如：

```text
nums = [1, 2, 3, 4]
```

每个位置右边乘积是：

```text
index 0: 2 * 3 * 4
index 1: 3 * 4
index 2: 4
index 3: 右边没有数字 -> 1
```

最后：

```text
answer[i] = leftProduct[i] * rightProduct[i]
```

## 6. 为什么可以用两遍扫描？

左边信息适合：

```text
从左往右维护
```

右边信息适合：

```text
从右往左维护
```

这和“固定哪个点与遍历方向”的思想是一致的。

如果当前位置需要左边累计结果：

```text
从左往右，左边已经见过。
```

如果当前位置需要右边累计结果：

```text
从右往左，右边已经见过。
```

所以这题自然会想到：

```text
一遍从左到右
一遍从右到左
```

## 7. 空间优化的思路

最直观的做法是开两个数组：

```text
left[]
right[]
```

分别存每个位置左边乘积和右边乘积。

但题目说输出数组不算额外空间。

所以可以进一步思考：

```text
能不能先把左边乘积存在 answer 里？
然后从右往左时，用一个变量维护右边乘积？
```

这就是空间优化的方向。

重点不是背代码，而是理解：

```text
当前答案需要左右两边的信息。
左边可以存在 answer 里。
右边可以用一个变量滚动维护。
```

## 8. 和 Prefix Sum 的关系

Prefix Sum 是：

```text
前缀累计和
```

这题是：

```text
前缀累计乘积
后缀累计乘积
```

它们背后的思想相同：

```text
把重复计算的累计结果提前维护起来。
```

只是运算从加法变成了乘法。

## 9. 这题真正要记住什么？

```text
1. answer[i] 不包括 nums[i] 自己
2. 不用除法
3. 每个位置的答案 = 左边乘积 * 右边乘积
4. 左边信息从左往右维护
5. 右边信息从右往左维护
6. 输出数组可以复用来降低额外空间
```

## 10. 记忆

```text
Product Except Self:
不是 total / nums[i]。
而是 left product * right product。
```

```text
左边靠 prefix。
右边靠 suffix。
```
