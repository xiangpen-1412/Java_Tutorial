# 02. House Robber

## 题目

给一个数组 `nums`，每个元素表示一间房子里的钱。

你不能偷相邻的两间房子，否则会报警。

题目要返回：

```text
在不偷相邻房子的前提下，最多能偷多少钱。
```
## 题号

```text
LeetCode 198. House Robber
```

## 1. 这题学到的核心

这题学到的是：

```text
DP 里每一步都可以看成一个选择。
```

对每一间房子来说，选择只有两个：

```text
偷当前房子
不偷当前房子
```

但是限制是：

```text
不能偷相邻的房子。
```

所以如果偷当前房子，上一间就不能偷。

## 2. dp 状态怎么定义

这题的状态可以定义成：

```text
res[i] = 只考虑从第 0 间到第 i 间房子时，最多能偷多少钱
```

注意：

```text
res[i] 不是一定偷第 i 间。
```

它表示的是：

```text
看到第 i 间为止的最大收益。
```

这个区别很重要。

## 3. 当前状态从哪里来？

走到第 `i` 间房子时，有两个选择。

### 选择 1：不偷当前房子

如果不偷第 `i` 间，那么最大收益就是前一间为止的最大收益：

```text
res[i - 1]
```

### 选择 2：偷当前房子

如果偷第 `i` 间，那么第 `i - 1` 间不能偷。

所以收益是：

```text
nums[i] + res[i - 2]
```

### 两者取最大

因此当前状态来自：

```text
max(偷当前, 不偷当前)
```

也就是：

```text
res[i] = Math.max(nums[i] + res[i - 2], res[i - 1])
```

## 4. 初始状态

最小情况要先处理。

如果只有一间房：

```text
直接返回 nums[0]
```

如果至少两间房：

```text
res[0] = nums[0]
res[1] = Math.max(nums[0], nums[1])
```

含义：

```text
只看第 0 间，最多就是偷第 0 间。
只看前 2 间，不能都偷，所以选金额更大的那间。
```

## 5. 为什么这是 Bottom-up DP？

这份写法是 bottom-up。

因为它是：

```text
先知道最小情况
再从左到右推出后面的结果
```

顺序是：

```text
res[0]
res[1]
res[2]
res[3]
...
res[n - 1]
```

每一步都只依赖已经算过的结果：

```text
res[i - 1]
res[i - 2]
```

## 6. 为什么数组比 HashMap 更合适？

之前也可以用 HashMap 存：

```text
index -> 当前最大收益
```

但这题的 index 是连续的：

```text
0, 1, 2, 3 ...
```

这种情况下，用数组更自然：

```text
res[i]
```

HashMap 更适合：

```text
key 不连续
状态比较复杂
需要用对象/组合状态做 key
```

这题用 HashMap 会有额外开销：

```text
hash 计算
Integer 装箱
map 查找
```

而数组直接按下标访问，更简单也更快。

## 7. 这题还能怎么优化？

当前数组写法空间是：

```text
O(n)
```

但状态转移只用到：

```text
res[i - 1]
res[i - 2]
```

所以理论上不需要保存整个数组。

可以只保留两个变量：

```text
前一个状态
前前一个状态
```

这叫：

```text
空间压缩
```

先用数组理解 DP 是对的。

等思路熟了，再考虑空间优化。

## 8. 这题的学习重点

这题不是为了记某段代码。

真正要记的是：

```text
1. dp[i] 表示“看到第 i 个位置为止的最优结果”
2. 当前房子只有偷和不偷两种选择
3. 偷当前，就不能偷前一个
4. 不偷当前，就继承前一个最优结果
5. 当前结果取两种选择的最大值
```
