# 00. 介绍 DP

## 1. DP 是什么？

DP 是 Dynamic Programming，中文叫动态规划。

它解决的问题通常有两个特点：

```text
1. 大问题可以拆成小问题
2. 小问题会被重复使用
```

DP 的核心不是背公式，而是：

```text
把小问题的答案存起来，再用小问题推出大问题。
```

## 2. 为什么需要 DP？

有些题可以用递归想出来，但是普通递归会重复算很多次。

比如一个问题：

```text
f(5) = f(4) + f(3)
f(4) = f(3) + f(2)
```

这里 `f(3)` 被重复算了。

如果 n 很大，重复计算会越来越多，最后超时。

DP 的做法是：

```text
f(3) 算过一次以后，就存起来。
下次再需要 f(3)，直接拿。
```

## 3. DP 的两种常见写法

### Top-down: 递归 + memo

也叫记忆化搜索。

```text
从大问题开始递归往下拆。
每算完一个小问题，就放进 memo。
下次遇到同一个问题，直接返回 memo 里的答案。
```

常见结构：

```text
function dfs(state):
    if memo 里有 state:
        return memo[state]

    if state 是基础情况:
        return 基础答案

    ans = 用更小的 state 算出来
    memo[state] = ans
    return ans
```

### Bottom-up: 迭代填表

也叫 tabulation。

```text
从最小的问题开始填。
用前面已经填好的答案，推出后面的答案。
```

常见结构：

```text
dp[base] = 基础答案

for state 从小到大:
    dp[state] = 用之前的 dp 推出来
```

## 4. 什么是初始状态？

初始状态就是：

```text
DP 开始推导之前，最小、最确定、不用再计算的答案。
```

可以把 DP 想成填表。

初始状态就是：

```text
一开始就能填好的格子。
```

比如走格子问题：

```text
dp[i][j] = 走到格子 (i, j) 的方法数
```

起点不用从别的地方推出来，所以：

```text
dp[0][0] = 1
```

这个就是初始状态。

没有初始状态，递推公式就没有地方开始。

## 5. DP 的四个问题

以后遇到 DP，先问这四个问题：

```text
1. dp[i] 表示什么？
2. 初始状态是什么？
3. 状态转移是什么？
4. 最后返回什么？
```

### dp[i] 表示什么？

这是最重要的一步。

比如：

```text
dp[i] = 到第 i 阶的方法数
dp[i] = 到第 i 天最多能赚多少钱
dp[i] = 以 index i 结尾的最大子数组和
```

定义错了，后面公式就会乱。

### 初始状态是什么？

就是一开始已经知道的基础答案。

比如：

```text
dp[0]
dp[1]
```

这些通常不用推。

### 状态转移是什么？

就是：

```text
当前 dp 怎么由前面的 dp 推出来？
```

比如：

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

### 最后返回什么？

题目问什么，就返回对应状态。

比如题目问第 n 个答案：

```text
return dp[n]
```

## 6. 怎么判断一道题可能是 DP？

看到这些信号可以考虑 DP：

```text
ways / 多少种方法
maximum / minimum
能不能到达
当前结果依赖前面的结果
普通递归会重复计算
```

但不是所有 maximum/minimum 都是 DP。

真正关键是：

```text
能不能用小问题推出大问题。
```

## 7. 记忆

```text
DP = 定义状态 + 初始状态 + 状态转移 + 返回答案
```

或者：

```text
先想 dp[i] 是什么，再想它从哪里来。
```
