# 03. Best Time to Buy and Sell Stock

## 题目

给一个数组 `prices`，`prices[i]` 表示第 `i` 天的股票价格。

你只能完成一次交易：

```text
先买一次
以后某一天再卖一次
```

题目要返回这次交易能获得的最大利润。如果无法赚钱，返回 `0`。
## 题号

```text
LeetCode 121. Best Time to Buy and Sell Stock
```

## 1. 这题学到的核心

这题最重要的不是代码，而是思路转变：

```text
不要固定买入日，然后每次往未来找最高卖出价。
```

那样容易变成：

```text
O(n^2)
```

真正要学的是：

```text
换一个观察方向，让需要的信息可以被维护。
```

## 2. 暴力思路

最直接的想法是：

```text
枚举买入日 buy
枚举 buy 后面的卖出日 sell
计算 prices[sell] - prices[buy]
取最大
```

这个思路能表达题目本质：

```text
找一对 buy < sell，让利润最大。
```

但是它慢在：

```text
每个 buy 都要重新往后找最好的 sell。
```

## 3. 固定买入日的问题

如果固定今天买：

```text
今天买的话，未来最高卖价是多少？
```

这个信息在右边。

如果从左往右扫，右边是未来，每次都要重新找。

所以会慢。

这时候有两种转变。

## 4. 转变 1：固定卖出日

反过来想：

```text
如果今天卖，我之前最低能以多少钱买入？
```

这个信息在左边。

从左往右扫时，左边已经看过了。

所以可以维护：

```text
到目前为止见过的最低价格
到目前为止能得到的最大利润
```

这不是死记 DP。

这是把问题变成：

```text
今天卖出的最好利润 = 今天价格 - 过去最低买入价
```

然后不断更新历史最大利润。

## 5. 转变 2：从右往左看

如果还是想固定买入日，也可以。

但要换方向：

```text
从右往左扫
```

这样走到某一天时，它右边的未来价格已经看过了。

可以维护：

```text
未来最高卖出价
最大利润
```

所以：

```text
固定卖出日 + 从左往右
固定买入日 + 从右往左
```

本质是一样的：

```text
把每次重新扫描，变成边走边维护。
```

## 6. 为什么它可以被看成 DP？

这题可以理解成 DP / Greedy / 状态维护。

DP 视角是：

```text
到第 i 天为止的最大利润
```

今天的最大利润来自两种情况：

```text
不在今天卖：最大利润还是之前的最大利润
在今天卖：今天价格 - 之前最低买入价
```

所以当前结果依赖过去维护的状态。

但这题状态很简单，不需要 dp 数组。

只需要变量：

```text
最低买入价
最大利润
```

## 7. 为什么不是 Sliding Window？

Sliding Window 适合：

```text
连续区间
窗口扩张/收缩
维护窗口里的条件
```

这题不是找一个连续区间。

它只找两个点：

```text
买入点
卖出点
```

中间的天数不作为窗口整体参与计算。

所以这题不适合用普通 sliding window 理解。

## 8. 为什么不是典型 Two Pointers？

典型 Two Pointers 经常依赖：

```text
数组有序
sum 太小 left++
sum 太大 right--
```

股票价格数组没有排序。

不能通过大小关系稳定移动左右指针。

所以它不是 3Sum 那种左右夹逼。

## 9. 这题真正要记住什么？

不要记答案。

记这个思考过程：

```text
1. 暴力是枚举 buy 和 sell。
2. 慢在每个 buy 都要往未来找最高 sell。
3. 如果固定 sell，只需要过去最低 buy。
4. 过去最低 buy 可以从左往右维护。
5. 如果固定 buy，也可以从右往左维护未来最高 sell。
```

## 10. 记忆

```text
固定哪一天，不是随便的。
```

```text
固定卖出日：
需要过去最低买入价。
从左往右。
```

```text
固定买入日：
需要未来最高卖出价。
从右往左。
```

```text
换方向的目的：
让未来信息变成已经见过的信息。
```
