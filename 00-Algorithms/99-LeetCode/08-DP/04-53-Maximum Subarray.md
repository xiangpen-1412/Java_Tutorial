# LeetCode 53 Maximum Subarray 做题复盘

题目一句话：给一个整数数组 `nums`，找一个连续子数组，使它的和最大，返回这个最大和。

例如：

```text
nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
答案是 6，对应子数组 [4, -1, 2, 1]
```

这篇笔记的重点不是背答案，而是理解：

```text
为什么你会想到滑动窗口？
为什么这道题不能按标准滑动窗口做？
这道题真正的算法思想是什么？
以后遇到类似题，怎么判断该用什么方法？
```

---

## 1. 你做题时的第一反应

你看到题目里有：

```text
continuous subarray
连续子数组
```

所以第一反应是：

```text
是不是可以用滑动窗口？
```

这个反应很正常，而且不算离谱。

因为滑动窗口确实经常处理：

```text
连续子数组
连续子串
一段区间
```

但是这里有一个关键误区：

```text
“连续”只是滑动窗口的必要条件，不是充分条件。
```

也就是说：

```text
能用滑动窗口的题，一般都在处理连续区间。
但处理连续区间的题，不一定都能用滑动窗口。
```

这道题就是一个典型反例。

---

## 2. 你真正犯的错误

你犯的核心错误不是“想到了滑动窗口”。

真正的问题是：

```text
你把“连续子数组”直接等同于“滑动窗口”。
```

但滑动窗口不是只靠 `left` 和 `right` 两个指针就成立。

滑动窗口必须能回答一个问题：

```text
什么时候移动 left？
```

比如标准滑动窗口通常是：

```text
right 向右扩张窗口
窗口不满足条件了，就移动 left
窗口满足条件时，更新答案
```

所以它一定需要一个明确的“窗口条件”。

例如：

```text
窗口长度必须等于 k
窗口和必须小于等于 target
窗口里不能有重复字符
窗口里最多只能有 k 种字符
```

但是 Maximum Subarray 这题里，你没有这样的条件。

题目只是说：

```text
找最大和
```

那问题来了：

```text
当前窗口和变小了，要不要移动 left？
遇到负数，要不要移动 left？
窗口和小于 0，要不要移动 left？
窗口和还是正数，但是中间有负数，要不要移动 left？
```

如果这些问题没有稳定答案，就说明它不是一个标准滑动窗口模型。

---

## 3. 为什么负数让滑动窗口失效？

滑动窗口很依赖一个东西：

```text
单调性
```

意思是窗口变化以后，状态变化是可预测的。

比如一个全是正数的数组：

```text
nums = [2, 3, 1, 2, 4, 3]
```

如果 `right` 往右走，窗口和一定变大。

如果 `left` 往右走，窗口和一定变小。

所以我们可以很放心地写：

```java
while (sum >= target) {
    // 更新答案
    sum -= nums[left];
    left++;
}
```

因为移动 `left` 的效果是确定的：`sum` 会减少。

但是这道题允许负数。

例如：

```text
nums = [5, -100, 6]
```

窗口从 `[5]` 扩张到 `[5, -100]`：

```text
sum 从 5 变成 -95
```

right 往右走，sum 反而变小了。

再看移动 left：

```text
[5, -100, 6] sum = -89
```

如果移掉 `5`：

```text
[-100, 6] sum = -94
```

sum 更小了。

如果移掉 `-100`：

```text
[6] sum = 6
```

sum 又变大了。

这说明：

```text
有负数时，移动 left 以后 sum 不一定变大，也不一定变小。
right 扩张以后 sum 也不一定变大。
```

所以你没有一个稳定规则来控制窗口。

这就是这道题不能按标准滑动窗口做的根本原因。

---

## 4. 你当时可能卡住的疑问

### 疑问一：遇到负数是不是就应该丢掉？

不是。

例如：

```text
[4, -1, 2, 1]
```

`-1` 是负数，但它应该保留。

因为这一整段的和是：

```text
4 + (-1) + 2 + 1 = 6
```

它是最优答案。

如果你因为看到 `-1` 就想把它丢掉，就会破坏连续性。

所以：

```text
单个负数不一定坏。
只要它前后的整体收益足够大，它就可以留在最优子数组里。
```

### 疑问二：那什么时候可以丢掉前面的部分？

不是看“有没有负数”，而是看：

```text
前面整段连续和是不是小于 0。
```

如果前面整段和是负数，比如：

```text
sum = -3
```

后面来了一个数字 `x`。

带上前面：

```text
-3 + x
```

不带前面：

```text
x
```

一定有：

```text
-3 + x < x
```

所以只要前面的连续和小于 0，它对后面一定是拖累。

这时可以从当前位置重新开始。

### 疑问三：这是不是也算滑动窗口？

它有一点像，但不是标准滑动窗口。

因为你没有维护一个明确的 `[left, right]`，也没有一个“窗口不合法就收缩”的过程。

这道题真正做的是：

```text
每走到一个位置，判断：
我要不要继承前面的连续和？
还是从当前数字重新开始？
```

这更像动态规划。

## 5. 这道题真正的算法思想

这道题的核心不是“维护窗口”，而是“做选择”。

当你遍历到 `nums[i]` 时，只需要问一个问题：

```text
如果子数组必须以 nums[i] 结尾，那么最大和是多少？
```

为什么要加上“必须以 nums[i] 结尾”？

因为这样问题就可以从前一个位置推过来。

对于 `nums[i]`，只有两个选择：

```text
选择 1：接在前面的连续子数组后面
选择 2：不接前面，从 nums[i] 自己重新开始
```

也就是：

```text
cur + nums[i]
nums[i]
```

取更大的那个：

```text
cur = max(nums[i], cur + nums[i])
```

这里的 `cur` 表示：

```text
以当前位置结尾的最大连续子数组和
```

再用一个 `ans` 记录整个过程中见过的最大值：

```text
ans = max(ans, cur)
```

这就是 Kadane 算法。

---

## 6. Kadane 算法背后的直觉

可以把 `cur` 理解成：

```text
我目前背着的一段连续和。
```

每来一个新数字 `num`，你都要决定：

```text
是继续背着前面的和，把 num 接上？
还是把前面的包袱扔掉，从 num 重新开始？
```

如果前面的 `cur` 是正数：

```text
cur + num > num
```

说明前面是帮助，应该接上。

如果前面的 `cur` 是负数：

```text
cur + num < num
```

说明前面是拖累，应该丢掉。

所以本质判断是：

```text
前面的连续和到底是帮助，还是拖累？
```

不是判断：

```text
当前数字是不是负数。
```

这个区别非常重要。

## 7. 用例子看思路变化

数组：

```text
[-2, 1, -3, 4, -1, 2, 1, -5, 4]
```

初始：

```text
cur = -2
ans = -2
```

走到 `1`：

```text
接前面：-2 + 1 = -1
重新开始：1
选择 1

cur = 1
ans = 1
```

这里说明：

```text
-2 是拖累，丢掉。
```

走到 `-3`：

```text
接前面：1 + (-3) = -2
重新开始：-3
选择 -2

cur = -2
ans = 1
```

虽然现在 `cur` 是负数，但它只是表示：

```text
如果必须以 -3 结尾，最好也只能做到 -2。
```

走到 `4`：

```text
接前面：-2 + 4 = 2
重新开始：4
选择 4

cur = 4
ans = 4
```

这里说明：

```text
前面的连续和是负数，丢掉，从 4 重新开始。
```

走到 `-1`：

```text
接前面：4 + (-1) = 3
重新开始：-1
选择 3

cur = 3
ans = 4
```

注意这里：

```text
-1 是负数，但没有丢掉。
```

因为接在 `[4]` 后面仍然比单独从 `-1` 开始好。

继续走：

```text
到 2:
cur = max(2, 3 + 2) = 5
ans = 5

到 1:
cur = max(1, 5 + 1) = 6
ans = 6
```

所以得到：

```text
[4, -1, 2, 1]
```

最大和：

```text
6
```

## 8. 代码

```java
class Solution {
    public int maxSubArray(int[] nums) {
        int cur = nums[0];
        int ans = nums[0];

        for (int i = 1; i < nums.length; i++) {
            cur = Math.max(nums[i], cur + nums[i]);
            ans = Math.max(ans, cur);
        }

        return ans;
    }
}
```

这段代码真正表达的是：

```text
cur = 以当前元素结尾的最大连续和
ans = 全局最大连续和
```

最关键的一句：

```java
cur = Math.max(nums[i], cur + nums[i]);
```

翻译成人话：

```text
我要么从 nums[i] 重新开始，
要么把 nums[i] 接到前面的最佳连续和后面。
哪个更大，就选哪个。
```

## 9. 一个容易写错的版本

有时你可能会写成：

```java
int sum = 0;
int ans = 0;
```

然后遍历累加。

这个写法有风险。

因为数组可能全是负数：

```text
nums = [-3, -2, -5]
```

正确答案是：

```text
-2
```

如果 `ans` 初始化成 `0`，最后会错误返回：

```text
0
```

但题目要求子数组至少包含一个元素，不能选择空数组。

所以：

```text
ans 要初始化为 nums[0]
cur 也要初始化为 nums[0]
```

这是这道题的一个重要边界。

## 10. 以后怎么判断是不是滑动窗口？

以后看到连续子数组，不要马上套滑动窗口。

先问自己四个问题：

```text
1. 题目有没有明确的窗口条件？
2. right 扩张以后，窗口状态是否有稳定变化？
3. left 收缩以后，窗口状态是否有稳定变化？
4. 我能不能清楚地说出什么时候移动 left？
```

如果能回答，比如：

```text
窗口长度超过 k，就移动 left
窗口里有重复字符，就移动 left
窗口和大于 target，就移动 left
窗口种类数超过 k，就移动 left
```

那大概率可以考虑滑动窗口。

如果你回答不出来：

```text
不知道什么时候移动 left
遇到负数不知道删不删
窗口和变大变小没有规律
```

那就不要硬套滑动窗口。

这道题就是：

```text
不知道什么时候移动 left
因为负数有时候该保留，有时候该丢
```

所以应该换思路。

## 11. 这道题以后应该怎么想？

遇到这种题：

```text
连续子数组
求最大和或最优值
数组里有正有负
```

优先想：

```text
动态规划
Kadane 算法
```

然后问：

```text
如果必须以当前位置结尾，最优答案是什么？
```

这句话是解题入口。

只要你能定义出：

```text
以 i 结尾的最大连续和
```

就能推出：

```text
dp[i] = max(nums[i], dp[i - 1] + nums[i])
```

再把 `dp` 数组优化成一个变量 `cur`，就得到最终代码。

## 12. 你的复盘总结

这道题真正要记住的不是代码，而是下面几句话：

```text
连续子数组不等于滑动窗口。

滑动窗口需要明确的窗口条件，也需要窗口变化有规律。

Maximum Subarray 里有负数，窗口和没有单调性，所以标准滑动窗口不好用。

负数不一定要丢掉，负数前缀才可能要丢掉。

如果前面的连续和小于 0，它一定会拖累后面的数，所以从当前数重新开始。

cur 表示以当前位置结尾的最大连续和。

ans 表示到目前为止见过的最大连续和。
```

最终记忆版：

```text
每到一个数，问：
我是接着前面，还是从这里重新开始？
```

代码记忆版：

```java
cur = Math.max(nums[i], cur + nums[i]);
ans = Math.max(ans, cur);
```
