# DP 已做题目总结：定义状态、写出转移、确认返回值

动态规划不是先创建一个 `dp[]`，而是把大问题表示成会重复出现的子问题，并保存子问题结果。当前笔记覆盖一维基础 DP、选择型 DP、状态压缩和连续最优状态。

## 2. 写 DP 前必须回答四个问题

```text
1. dp[i] 表示什么？
2. 初始状态是什么？
3. 状态如何从更小状态转移？
4. 最后返回哪个状态？
```

如果第一问没有一句完整定义，后面的下标和转移通常都会混乱。

## 3. 两种实现方式

### Top-down：递归 + memo

从目标问题向下拆分；遇到已经计算过的状态就直接返回缓存。

```text
优点：接近递归定义
风险：忘记缓存、base case 错误、递归栈开销
```

### Bottom-up：迭代填表

先写已知的最小状态，再按照依赖顺序计算更大状态。

```text
优点：执行顺序明确，通常没有递归栈
关键：遍历方向必须保证依赖状态已经计算
```

## 4. Climbing Stairs：方案计数型

到第 `i` 阶的最后一步只能来自：

```text
i - 1 阶走 1 步
i - 2 阶走 2 步
```

两类方案互不重叠，因此相加。

它训练的是：从“最后一步来自哪里”推出状态转移。

## 5. House Robber：选择与不选择

处理第 `i` 间房时只有两种互斥决定：

```text
不偷当前：沿用前一个最优结果
偷当前：当前价值 + 隔一个位置之前的最优结果
```

所以状态来自二者最大值。

这类题的识别信号：

```text
每个位置可选或不选
相邻选择互相冲突
要求最大收益
```

## 6. Best Time to Buy and Sell Stock：维护历史最优信息

卖出价位于当前天时，最佳买入价一定是之前出现过的最低价格。

遍历时维护：

```text
minPrice：到当前为止的最低买价
maxProfit：到当前为止的最大利润
```

每个价格到来时，先用历史最低价计算可能利润，再更新全局结果。

这题说明 DP 不一定必须显式创建数组；只要当前状态只依赖少量历史信息，就可以状态压缩为变量。

## 7. Maximum Subarray：连续状态只能接或重开

定义：

```text
current = 以当前元素结尾的最大子数组和
```

加入当前元素时只有两种可能：

```text
从当前元素重新开始
接在前一个以相邻位置结尾的最优区间后面
```

全局最大值不能只看最后一个 `current`，因为最佳区间可能更早已经结束，所以另行维护 `best`。

## 8. 状态压缩

如果 `dp[i]` 只依赖：

```text
dp[i - 1]
dp[i - 2]
```

则不一定保留完整数组，可以只保存最近两个状态。

但优化空间之前应先写清楚完整状态定义和转移，否则变量含义容易丢失。

## 9. 选择方法

| 题目信号 | 当前笔记中的模型 |
| --- | --- |
| 到达某位置有多少种方法 | 方案计数 DP |
| 每个位置选或不选，选择有冲突 | 选择型 DP |
| 当前答案依赖历史最低/最高状态 | 维护压缩状态 |
| 最大连续区间，必须以当前位置结尾 | Kadane / 连续状态 DP |

## 10. 常见错误

```text
没有定义 dp[i] 就直接写公式；
初始状态与状态含义不一致；
普通递归重复计算导致超时；
memo 使用默认值时，无法区分“没算过”和合法结果；
House Robber 的“偷当前”错误地加上相邻状态；
Maximum Subarray 只返回最后位置状态，没有维护全局最大值；
过早状态压缩，变量更新顺序覆盖了仍需使用的旧值。
```

## 11. 记忆主线

```text
先定义状态，再写转移。
计数问题看最后一步来自哪里。
选择问题比较选与不选。
连续问题关注“必须以当前位置结尾”的状态。
只依赖少量历史状态时再做空间压缩。
```
