# Longest Substring：这道题真正要学什么

## 题目

给一个字符串 `s`，找出其中最长的、不包含重复字符的连续子字符串长度。

重点是：

```text
substring 必须连续。
返回长度，不是返回字符串本身。
```
## 1. 核心思想：滑动窗口

这道题真正要学的是：

> 维护一个当前合法的连续区间，而不是把字符串切成很多段再比较。

这里的“合法区间”就是：

```text
当前窗口里没有重复字符。
```

如果窗口合法，就继续往右扩大；如果窗口出现重复，就从左边收缩，直到重新合法。

---

## 2. 四个变量的作用

```text
left  ：窗口左边界
right ：窗口右边界
set   ：当前窗口里的字符
max   ：历史最大长度
```

它们的分工是：

```text
right 负责加入新字符。
left 负责在重复时移动窗口。
set 负责判断有没有重复。
max 负责记录最长结果。
```

重点：

```text
set 不是存所有出现过的字符。
set 只存当前窗口里的字符。
```

---

## 3. 遇到重复时怎么处理

当 `right` 指向一个字符时：

如果这个字符不在 `set` 里：

```text
加入 set，扩大窗口，更新 max。
```

如果这个字符已经在 `set` 里：

```text
从 left 开始删除字符。
left 一直往右走。
直到 set 里没有当前这个重复字符。
然后再把当前字符加入窗口。
```

所以这题不是“重复了就重新开始”，而是：

```text
重复了就调整左边界，让窗口重新合法。
```

---

## 4. 和我原来分段思路的区别

分段思路是：

```text
找第一段不重复的。
遇到重复后，再开第二段。
比较哪一段更长。
```

这个思路容易复杂，因为你要判断：

```text
新段从哪里开始？
旧段哪些字符还能保留？
什么时候清空？
什么时候替换？
```

滑动窗口不关心“旧段”和“新段”。

它只关心一件事：

```text
当前窗口是否合法。
```

不合法就移动 `left` 修复，合法就继续移动 `right` 扩大。

---

## 5. 这道题背后的题型模式

这题属于滑动窗口。

以后看到这些关键词，可以先往滑动窗口想：

```text
最长 / 最短
连续子串 / 连续子数组
满足某个条件
```

这道题就是：

```text
最长 + 连续子串 + 不重复字符
```

所以可以转化成：

```text
维护一个没有重复字符的窗口，并让它尽量变长。
```

---

## 6. 最后要记住的思路

不要死记代码。

这道题真正要记的是：

```text
right 扩大窗口。
set 判断是否合法。
如果重复，left 收缩窗口。
每次窗口合法后，更新 max。
```

一句话总结：

> 当题目要求“最长/最短的连续区间，并且区间要满足某个条件”时，优先考虑滑动窗口。
