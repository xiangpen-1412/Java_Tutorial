# LeetCode 15：3Sum 背后的知识

## 1. 题目核心

3Sum 要求从数组中找出所有不重复的三元组，使得：

```text
a + b + c = 0
```

它和 Two Sum 的区别是：

```text
Two Sum：找两个数
3Sum：先固定一个数，再在剩下的范围里找另外两个数
```

所以 3Sum 可以理解成：

```text
固定一个数 + Two Sum II
```

---

## 2. 为什么要排序

这道题通常先排序：

```java
Arrays.sort(nums);
```

排序有两个作用：

```text
1. 让双指针可以根据 sum 的大小移动
2. 让重复数字挨在一起，方便去重
```

排序不是为了改变答案，而是为了让数组有结构。

有序以后，才能判断：

```text
sum 太小，left++
sum 太大，right--
```

---

## 3. 为什么不能直接 distinct

这道题不能提前删除重复数字。

比如：

```text
[-1, -1, 2]
```

答案需要两个 `-1`：

```text
[-1, -1, 2]
```

如果直接 `distinct()`，就会把一个 `-1` 删除，导致答案丢失。

所以这道题要区分：

```text
重复数字可以使用
重复答案不能返回
```

正确思路是：

```text
保留原来的数字数量
但在生成答案时跳过重复组合
```

---

## 4. 固定一个数，把问题降级

固定 `nums[i]` 后，原问题：

```text
nums[i] + nums[left] + nums[right] = 0
```

可以变成：

```text
nums[left] + nums[right] = -nums[i]
```

这就是把 3Sum 降级成 Two Sum。

所以核心结构是：

```text
外层固定一个数
内层用双指针找另外两个数
```

---

## 5. 去重是这道题的重点

3Sum 难点不只是找到答案，而是不能返回重复三元组。

所以需要两层去重：

```text
1. 外层 i 去重：避免固定同一个值多次
2. 内层 left/right 去重：避免重复加入相同 pair
```

比如：

```text
[-1, -1, 0, 1, 1]
```

虽然不同下标可以组成多次：

```text
[-1, 0, 1]
```

但结果里只能返回一次。

---

## 6. 这道题背后真正要学什么

这道题主要训练这些能力：

```text
1. 把复杂问题拆成简单问题
2. 理解 3Sum = 固定一个数 + Two Sum
3. 知道排序可以制造结构
4. 用双指针减少暴力枚举
5. 区分“重复数字”和“重复答案”
6. 学会在有重复元素的题里做去重
```

---

## 一句话总结

LeetCode 15 真正要学的是：

> 通过排序制造结构，固定一个数把 3Sum 降级成 Two Sum，再用双指针找答案，同时处理重复结果。
