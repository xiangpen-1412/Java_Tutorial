# Array 已做题目总结：先识别数据性质，再选择指针或哈希结构

这份总结只整理当前 `01-Array` 中已经存在的题目。数组只是输入形式，真正决定解法的是：是否有序、是否要求原地修改、是否研究连续区间、是否需要快速查找对应值。

## 1. 已覆盖的题型

| 题目 | 题目类型 | 主要方法 | 核心判断 |
| --- | --- | --- | --- |
| [[01-1-Two Sum]] | 无序数组找两数和 | HashMap | 查找 `target - current` 是否出现过 |
| [[02-125-Valid Palindrome]] | 两端字符配对 | 相向双指针 | 跳过无效字符后比较左右字符 |
| [[03-3-Longest Substring Without Repeating Characters]] | 连续区间满足无重复 | 滑动窗口 | 右边扩张，冲突时移动左边 |
| [[04-283-Move Zeroes]] | 原地稳定搬移 | 读写指针 | 把非零元素依次写到前面 |
| [[05-167-Two Sum II - Input Array Is Sorted]] | 有序数组找两数和 | 相向双指针 | 和太小移左边，和太大移右边 |

## 2. HashMap：从“向后寻找”改成“查询已知信息”

### Two Sum

遍历到 `current` 时，需要的另一个数是：

```text
need = target - current
```

暴力解法每次向后扫描；HashMap 把已经见过的值及下标保存起来，使查询平均为 `O(1)`。

识别信号：

```text
找两个元素满足某种对应关系
需要返回原下标
数组没有可利用的有序性
```

### Group Anagrams

这题不是查补数，而是用 HashMap 分组：

```text
key   = 字符串的标准形式
value = 具有相同标准形式的字符串列表
```

标准形式可以来自排序结果或字符频率。真正的模式是：

> 先为每个对象提取能代表所属类别的 key，再按照 key 放入不同桶中。

## 3. 两类双指针必须分清

### 相向双指针

```text
left  →       ← right
```

适用于从两端比较，或者利用有序性排除答案。

当前题目：

```text
125：比较左右有效字符
167：根据当前和的大小移动一侧
15：固定一个数后，在剩余有序区间找两数和
```

### 同向读写指针

```text
write →
read  →→→
```

`read` 负责扫描，`write` 表示下一个有效元素应写入的位置。

当前题目：

```text
283：读到非零元素就写入前部，最后补零
26：读到与上一个有效值不同的元素才写入
```

这类题的重点不是“删除数组元素”，而是维护一个有效前缀：

```text
nums[0 ... write - 1] 始终是已经处理好的答案区域
```

## 4. 滑动窗口：维护一个连续区间

[[03-3-Longest Substring Without Repeating Characters]] 研究的是连续子串，而不是任意选择字符。

窗口通常表示：

```text
[left, right] = 当前仍满足条件的连续区间
```

处理顺序：

```text
right 加入新元素
如果窗口违反条件，移动 left 恢复合法
合法后更新答案
```

判断是否应该想到滑动窗口：

```text
题目研究子数组 / 子串
区间必须连续
右端扩张后，可以通过移动左端恢复条件
```

## 5. 排序的作用不只是让结果有序

[[06-15-3Sum]] 中排序提供三种能力：

1. 可以根据和的大小决定移动哪一侧；
2. 相同元素相邻，方便跳过重复答案；
3. 固定一个数后，把三数问题降为标准两数问题。

去重必须发生在产生重复选择的位置：

```text
固定点重复 → 跳过重复固定点
找到答案后 → 跳过左右两端的重复值
```

## 6. 一张选择方法的判断表

| 题目信号 | 优先考虑 |
| --- | --- |
| 无序数组中快速找对应值 | HashMap |
| 按共同特征把元素分组 | 标准化 key + HashMap |
| 有序数组中找两数关系 | 相向双指针 |
| 从两端比较字符或元素 | 相向双指针 |
| 原地删除、搬移、保留有效元素 | 读写指针 |
| 最长/最短连续子串，窗口能逐步恢复合法 | 滑动窗口 |
| 三数和且要求去重 | 排序 + 固定一点 + 双指针 |

## 7. 当前题目中常见错误

```text
看到数组就只想到 for 循环，没有先看“有序、连续、原地”等性质；
读写指针中混淆“当前读取位置”和“下一个写入位置”；
滑动窗口遇到冲突后只移动一次 left，没有持续恢复合法；
3Sum 只对最后结果 distinct，没有在选择阶段去重；
Two Sum 排序后忘记题目要求原始下标；
Group Anagrams 的 key 不能稳定代表同一类别。
```

## 8. 记忆主线

```text
无序查对应值      → HashMap
有序两端可排除    → 相向双指针
原地压缩有效区域  → 读写指针
连续区间动态变化  → 滑动窗口
多个元素组合      → 固定一个点，把问题降级
对象需要分类      → 提取统一 key 后分组
```
