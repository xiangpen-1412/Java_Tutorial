# LeetCode 167：Two Sum II 背后要学什么

## 1. 题目核心

这道题给的是一个已经排好序的数组，让你找两个数，使它们的和等于 `target`。

它和普通 Two Sum 最大的区别是：

```text
数组已经有序
```

这个条件非常关键。

---

## 2. 这道题真正想训练什么

这道题不是单纯考“找两个数”。

它真正想训练的是：

> 当数据已经有序时，不要急着暴力枚举，要学会利用顺序关系减少无意义尝试。

如果数组无序，可能需要 HashMap。

但如果数组有序，就应该优先思考：

```text
能不能用左右指针？
能不能根据大小关系排除一部分情况？
```

---

## 3. 从暴力思路里学到什么

最自然的想法是：

```text
固定一个数
再去后面一个一个找另一个数
```

这个思路能理解，但问题是：

```text
每次只试一个组合
没有利用数组有序这个条件
```

所以这道题要提醒我：

> 暴力解法不是错，但写完暴力思路后，要继续问：哪里重复了？题目条件能不能帮我少试一些？

---

## 4. 有序数组带来的信息

数组有序意味着：

```text
左边的数更小
右边的数更大
```

所以当两个数的和：

```text
太大时，可以尝试让大的一边变小
太小时，可以尝试让小的一边变大
```

这就是双指针思想的来源。

它不是凭空想到的，而是从“有序”这个条件推出来的。

---

## 5. 这道题背后的 pattern

以后看到类似条件：

```text
有序数组
找两个数
要求 O(1) 额外空间
```

就要优先想到：

```text
双指针
```

而不是第一反应就开 HashMap 或者双层循环。

---

## 6. 我应该记住什么

这道题最重要的收获不是代码，而是思维方式：

```text
不要只想着把所有组合试一遍。
要先看题目给了什么结构。
如果结构能帮我排除情况，就要利用它。
```

---

## 一句话总结

LeetCode 167 真正要学的是：

> 有序数组里的 Two Sum，不应该暴力枚举所有组合，而应该利用大小关系，用双指针逐步排除不可能的情况。
