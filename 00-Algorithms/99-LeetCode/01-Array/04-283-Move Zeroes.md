# 283. Move Zeroes：思路总结

## 1. 题目真正要求

这道题不是简单排序，而是要求：

```text
把所有 0 移到最后
非 0 元素的相对顺序不能变
必须原地修改数组
```

比如：

```text
[0, 1, 0, 3, 12]
```

非 0 元素是：

```text
1, 3, 12
```

所以结果必须是：

```text
[1, 3, 12, 0, 0]
```

---

## 2. 我的原始思路

我一开始容易想成：

```text
看到 0，就把 0 往后移动
然后让后面的元素整体往前挪
```

这个思路很自然，但会变复杂。

因为每移动一个 0，后面的很多元素都可能要跟着移动。

---

## 3. 更好的思路：反过来处理非 0

不要想着怎么把 0 推到后面。

应该反过来想：

```text
把所有非 0 元素按原顺序写到前面
剩下的位置自然填 0
```

比如：

```text
[0, 1, 0, 3, 12]
```

先把非 0 写到前面：

```text
[1, 3, 12, ?, ?]
```

然后剩下的位置补 0：

```text
[1, 3, 12, 0, 0]
```

---

## 4. 核心变量：读写指针

这题可以理解成两个指针：

```text
i      ：负责从左到右扫描数组
insert ：负责记录下一个非 0 应该写到哪里
```

流程是：

```text
如果 nums[i] 不是 0
就把 nums[i] 写到 nums[insert]
然后 insert 往后移动
```

扫描结束后：

```text
从 insert 开始到数组末尾，全部填 0
```

---

## 5. 这题背后考察的思想

这题真正考察的是：

```text
原地数组处理
读写指针
稳定保留元素顺序
换视角处理问题
```

最重要的思维变化是：

```text
不要追着 0 移动。
而是把需要保留的非 0 元素按顺序写到前面。
```

---

## 6. 以后遇到类似题怎么想

如果题目要求：

```text
删除某类元素
移动某类元素
保留某类元素顺序
原地修改数组
```

可以先想：

```text
能不能用一个 write/insert 指针，
把我要保留的元素按顺序写到前面？
```

---

## 7. 一句话总结

这道题不要记成“怎么移动 0”。

应该记成：

```text
把非 0 按原顺序压到前面，剩下位置补 0。
```

核心模式是：

```text
读指针扫描元素，写指针放置有效元素。
```
