# 125. Valid Palindrome：思路总结

## 题目

给一个字符串 `s`，判断它是不是回文。

判断时要忽略：

```text
大小写
非字母数字字符
```

也就是说，只比较有效的字母和数字，并且从左往右读和从右往左读要一样。
## 1. 我的原始思路

我的第一反应是：

```text
先清洗字符串
再复制/反转一份
然后比较原字符串和反转字符串是否一样
```

这个思路能做，但问题是：

```text
多构造了一个副本
多做了一次完整转换
思考方式偏“先变形，再处理”
```

---

## 2. 更好的思路：直接局部比较

回文的本质不是“反转后一样”，而是：

```text
第一个字符 == 最后一个字符
第二个字符 == 倒数第二个字符
第三个字符 == 倒数第三个字符
```

所以不需要真的创建 reversed 字符串。

直接用两个指针：

```text
left  从左边开始
right 从右边开始
```

每次比较：

```text
s[left] 和 s[right]
```

不一样就返回 false；一样就继续往中间走。

---

## 3. 这题真正要学的思想

这题训练的是：

```text
不构造副本，直接在原数据上比较局部位置。
```

也就是：

```text
能用指针模拟的东西，就不要急着创建新结构。
```

比如：

```text
想反转比较 → 可以用左右指针直接比较
想截取一段 → 可以用 left/right 维护窗口
想生成完整结果 → 先想能不能边走边处理
```

---

## 4. 和之前题目的共同问题

我之前的思路经常是：

```text
先把数据转换成另一个完整形式
然后再处理
```

比如：

```text
2. Add Two Numbers：
想先把链表转成完整数字。

3. Longest Substring：
想先切成 old sequence / new sequence 再比较。

4. Valid Palindrome：
想先创建 reversed 副本再比较。
```

更好的算法思路是：

```text
顺着原数据结构处理。
每一步只处理当前需要的位置。
用指针或状态变量减少不必要的转换。
```

---

## 5. 以后怎么改进

做题时先问自己：

```text
1. 我真的需要创建一个新数组/新字符串吗？
2. 能不能直接在原数据上用指针比较？
3. 当前这一步真正需要的信息是什么？
4. 有没有办法边遍历边处理？
```

看到这些关键词时，优先想到双指针：

```text
回文
反转
首尾比较
左右两端
有序数组
原地处理
```

---

## 6. 一句话总结

这题不要记成“清洗字符串 + reverse 比较”。

应该记成：

```text
回文 = 左右对称。
左右对称 = left/right 双指针直接比较。
```

核心改进方向：

```text
少构造副本，多利用指针；
少整体转换，多局部处理。
```
