# Heap / PriorityQueue 已做题目总结：持续维护当前最重要的候选

堆不负责把所有元素完全排序，而是让当前最小值或最大值始终能快速取得。当前笔记覆盖 Top K 和数据流中位数两个核心模型。

| [[01-215-Kth Largest Element in an Array]] | 第 k 大 | 大顶堆或大小为 k 的小顶堆 | 堆中保留当前最大的 k 个元素 |

## 2. PriorityQueue 提供什么能力？

Java 默认：

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
```

是小顶堆，堆顶是最小值。

大顶堆：

```java
PriorityQueue<Integer> maxHeap =
        new PriorityQueue<>(Collections.reverseOrder());
```

主要操作：

```text
offer：O(log n)
poll：O(log n)
peek：O(1)
```

堆只保证堆顶正确，不保证遍历顺序整体有序。

## 3. Kth Largest：只保留与答案相关的 k 个元素

使用大小为 `k` 的小顶堆：

```text
每个数加入堆
如果 size > k，删除当前最小值
```

遍历结束后，堆中是当前最大的 `k` 个元素；其中最小的那个就是整体第 `k` 大，因此答案位于堆顶。

不变量：

```text
堆的大小不超过 k
堆中始终保留已经看过元素中的最大 k 个
```

相比完整排序，它只维护与答案有关的部分。

## 4. 数据流中位数：把数据分成左右两半

维护两个堆：

```text
left：大顶堆，保存较小的一半，堆顶是左半最大值
right：小顶堆，保存较大的一半，堆顶是右半最小值
```

两个不变量：

### 顺序不变量

```text
left 中所有元素 <= right 中所有元素
```

### 数量不变量

```text
两边大小差不超过 1
```

只要两个不变量成立，中位数就只与两个堆顶有关：

```text
总数为奇数 → 较大一侧的堆顶
总数为偶数 → 两个堆顶的平均值
```

## 5. 插入新元素的思考顺序

```text
先决定新值属于较小一半还是较大一半
再通过移动堆顶恢复数量平衡
最后根据堆大小读取中位数
```

具体写法可以不同，但结束后必须重新验证两个不变量，而不是只看某一个测试用例。

## 6. 为什么单堆不能直接维护中位数？

单个小顶堆只方便取得最小值，单个大顶堆只方便取得最大值。中位数位于排序后的中间，因此需要同时快速取得：

```text
较小一半中的最大值
较大一半中的最小值
```

双堆正好让这两个边界都位于堆顶。

## 7. 选择方法

| 题目信号 | 优先考虑 |
| --- | --- |
| 只关心最大/最小元素 | 单个堆 |
| 只保留最大的 k 个 | 大小为 k 的小顶堆 |
| 只保留最小的 k 个 | 大小为 k 的大顶堆 |
| 数据持续加入且反复查询中位数 | 双堆 |

## 8. 常见错误

```text
误以为 PriorityQueue 遍历结果整体有序；
第 k 大错误地使用大小为 k 的大顶堆却仍按小顶堆逻辑维护；
忘记把堆大小限制为 k；
双堆只平衡数量，没有保证左右大小关系；
初始堆为空时直接 peek；
偶数中位数使用整数除法，丢失小数；
两个 int 堆顶先相加可能溢出，再转换 double 已经太晚。
```

## 9. 复杂度

### Kth Largest

```text
时间：O(n log k)
空间：O(k)
```

### Median from Data Stream

```text
每次插入：O(log n)
查询中位数：O(1)
总空间：O(n)
```

## 10. 记忆主线

```text
堆用于持续维护“当前最重要的候选”，不是完整排序。
第 k 大 → 保留最大的 k 个，堆顶是其中最小者。
中位数 → 左半大顶堆 + 右半小顶堆。
双堆先保证左右顺序，再保证数量平衡。
```
