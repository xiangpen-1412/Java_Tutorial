栈适合处理“后出现的数据先与最近的未完成数据配对”。普通栈强调后进先出；单调栈进一步利用大小关系，批量删除以后不可能成为答案的候选项。

## 2. 普通栈：处理嵌套和最近配对

括号题中，右括号必须与最近尚未匹配的左括号配对，因此使用后进先出：

```text
左括号 → 入栈
右括号 → 检查栈顶是否匹配
结束后 → 栈必须为空
```

只统计括号数量不够，因为顺序和类型都必须正确。

## 3. 辅助栈：让额外查询保持 O(1)

Min Stack 不应在每次 `getMin()` 时扫描全部元素。辅助栈保存当前位置对应的最小值。

必须正确处理重复最小值：

```text
push 新值时使用 <=，或者每层都同步保存当前最小值
pop 数据时同步恢复上一层最小值
```

关键不是一定使用两个栈，而是维护不变量：

> 栈的每个状态都能立即知道当前最小值。

## 4. 单调栈解决什么问题？

典型问题：

```text
某元素左边/右边第一个更大或更小的元素在哪里？
某元素作为最大值或最小值能够影响多大范围？
```

处理当前元素时，如果栈顶已经被当前元素确定答案，就持续弹出：

```java
while (!stack.isEmpty() && 当前元素满足栈顶的答案条件) {
    int index = stack.pop();
    // 计算 index 的答案
}
```

每个下标最多入栈一次、出栈一次，因此总时间通常为 `O(n)`。

## 5. 栈里存值还是下标？

当前笔记大多数题需要距离、位置或修改答案数组，所以应保存下标：

```text
值：nums[index]
距离：currentIndex - index
答案位置：answer[index]
```

901 可以保存“价格 + 已压缩跨度”，因为它需要在线计算连续天数。

## 6. 当前题目的边界方向

```text
739：向右找第一个严格更大
503：环形数组中向右找第一个严格更大
1475：向右找第一个小于等于
901：向左找阻止连续跨度的更高价格
907：找元素作为最小值时的左右边界
84：找柱子作为最低高度时能向左右扩张的范围
```

单调递增还是递减不能只背名字，应先写清楚：

```text
谁在等待答案？
当前元素满足什么条件时，可以解决栈顶？
相等时应该弹出还是保留？
```

## 7. 环形数组怎么处理？

503 不需要真正复制数组，可以遍历 `2n` 个位置，并用：

```java
int index = i % n;
```

模拟绕回开头。第二遍只负责帮助第一遍中尚未找到答案的元素，避免重复加入无意义候选。

## 8. 从“找边界”到“贡献法”

907 和 84 都不是只找下一个元素，而是确定一个元素可以负责的完整范围。

### 907

对每个元素计算：

```text
作为子数组最小值的左侧选择数
× 右侧选择数
× 当前值
```

相等元素的一侧必须严格、另一侧必须允许相等，避免同一个子数组重复归属。

### 84

每根柱子作为矩形最低高度时：

```text
面积 = 高度 × 可扩张宽度
```

遇到更低柱子时，弹出的柱子确定了右边界；弹出后的新栈顶提供左边界。

## 9. 选择方法

| 题目信号 | 优先考虑 |
| --- | --- |
| 嵌套结构、最近匹配 | 普通栈 |
| push/pop 外还要求 O(1) 极值 | 辅助栈 |
| 左右第一个更大/更小 | 单调栈 |
| 环形 next greater | 单调栈 + `i % n` |
| 每个元素控制的连续范围 | 单调栈边界 |
| 所有子数组极值贡献之和 | 单调栈 + 贡献法 |

## 10. 常见错误

```text
while 前没有判断栈空；
需要距离却只保存值；
把“更大”与“更大或等于”的条件写反；
环形遍历第二遍错误地直接访问 nums[i]；
相等元素左右边界采用相同规则，导致重复计数；
弹栈后宽度的左边界计算少一或多一；
误以为嵌套 while 会变成 O(n²)，忽略每个元素最多弹出一次。
```

## 11. 记忆主线

```text
普通栈：最近的未完成任务先解决。
单调栈：删除不可能再成为候选的元素。
下一个更大/更小：当前元素解决栈顶。
范围贡献：弹栈时左右边界同时确定。
```
