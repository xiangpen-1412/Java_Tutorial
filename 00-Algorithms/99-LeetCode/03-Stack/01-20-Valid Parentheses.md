# LeetCode 20：有效括号背后的知识  
  
## 题目在考什么  
  
给你一个只包含括号的字符串，判断括号是否合法。  
  
合法的意思是：  
  
```text  
左括号和右括号数量要对应  
右括号必须关闭最近的那个左括号  
括号顺序不能乱  
```  
  
比如：  
  
```text  
()      合法  
([])    合法  
([)]    不合法  
```  
  
这道题真正考的是 Stack 的使用。  
  
---  
  
## 背后的核心知识：Stack  
  
Stack 叫做栈。  
  
它的特点是：  
  
```text  
后进先出  
Last In, First Out  
LIFO  
```  
  
可以理解成一摞盘子：  
  
```text  
最后放上去的盘子，最先被拿走。  
```  
  
---  
  
## 为什么括号匹配适合用 Stack  
  
括号匹配有一个规律：  
  
> 最后出现的左括号，必须最先被匹配。  
  
比如：  
  
```text  
({[]})  
```  
  
左括号出现顺序是：  
  
```text  
(  
{  
[  
```  
  
但是关闭的时候，必须先关闭 `[`，再关闭 `{`，最后关闭 `(`。  
  
这正好符合 Stack 的后进先出。  
  
---  
  
## Stack 在这道题里的作用  
  
遇到左括号，就放进栈里。  
  
遇到右括号，就检查栈顶是不是对应的左括号。  
  
如果匹配，说明这一对括号合法。  
  
如果不匹配，说明顺序错了。  
  
如果最后栈为空，说明所有括号都被正确匹配了。  
  
---  
  
## 为什么不能只数数量  
  
比如：  
  
```text  
([)]  
```  
  
这里 `(` 和 `)` 数量一样，`[` 和 `]` 数量也一样。  
  
但是它不合法。  
  
因为关闭顺序错了。  
  
所以这道题不是只看数量，而是看：  
  
> 当前右括号能不能匹配最近的左括号。  
  
---  
  
## Java 里的 Stack 写法  
  
Java 里现在更推荐用 `Deque` 来当栈：  
  
```java  
Deque<Character> stack = new ArrayDeque<>();  
```  
  
常用操作：  
  
```java  
stack.push(c);   // 入栈  
stack.pop();     // 出栈  
stack.peek();    // 看栈顶  
stack.isEmpty(); // 判断是否为空  
```  
  
---  
  
## Queue 和 Deque 顺带记一下  
  
### Queue  
  
Queue 是普通队列。  
  
特点是：  
  
```text  
先进先出  
First In, First Out  
FIFO  
```  
  
像排队买饭：  
  
```text  
先来的人先走。  
```  
  
### Deque  
  
Deque 是双端队列。  
  
特点是：  
  
```text  
头部和尾部都可以进出。  
```  
  
所以它比普通 Queue 更灵活。  
  
Java 里常用 `Deque` 做两件事：  
  
```text  
1. 当普通队列用  
2. 当 Stack 用  
```

## 一句话总结  
  
LeetCode 20 的核心是：  
  
> 用 Stack 记录还没被匹配的左括号，遇到右括号时检查它是否匹配最近的左括号。
